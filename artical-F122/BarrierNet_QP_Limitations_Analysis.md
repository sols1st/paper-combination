# BarrierNet + QP 为何无法实现百分百安全：局限性深度分析

> 摘要：BarrierNet 通过在神经网络末端添加可微分 QP（二次规划）安全滤波层，利用 CBF（Control Barrier Function）约束在运行时过滤不安全控制动作。这一机制大幅提升了安全性，但仍存在多个根本性局限，使其无法达到 100% 的安全保证。本文从数学本质出发，系统性地分析这些局限的来源，并讨论 SafePVC 的 SBC 如何从概率层面进行补充。

## 1. 引言：QP 安全滤波的承诺与现实

### 1.1 BarrierNet 的安全承诺

BarrierNet 的核心机制是在每个时间步求解以下二次规划：

$$u_t^* = \arg\min_{u \in \mathbb{R}^n} \quad \frac{1}{2}\|u\|^2 + q_t^T u$$

$$\text{s.t.} \quad G(s_t, p_t) \, u \leq h(s_t, p_t)$$

其中约束 $Gu \leq h$ 由 CBF 条件导出，理论上保证安全集的前向不变性：

$$b(s_t) \geq 0 \implies b(s_{t+1}) \geq 0$$

**承诺**：只要当前状态安全（$b(s_t) \geq 0$），经过 QP 过滤后的控制动作 $u_t^*$ 将确保下一时刻状态也安全。

### 1.2 现实：承诺存在多处"缝隙"

然而，这一承诺建立在一系列理想化假设之上。当这些假设在实际系统中不成立时，安全保证就会出现裂缝。以下逐一分析这些假设及其不成立时的后果。

---

## 2. 局限一：连续时间理论与离散时间执行的裂缝

### 2.1 CBF 理论的连续时间基础

CBF 的前向不变性证明建立在**连续时间**系统之上：

$$\dot{s} = f(s) + g(s)u$$

CBF 条件为：

$$\dot{b}(s) + \alpha(b(s)) \geq 0 \quad \implies \quad b(s(t)) \geq 0, \; \forall t \geq 0$$

这是一个**连续时间**的结论——它保证的是连续轨迹上每一个时刻的安全性。

### 2.2 实际系统的离散时间执行

然而，数字控制器只能以离散时间步长 $\Delta t$ 执行：

$$u(t) = u_k, \quad \forall t \in [k\Delta t, (k+1)\Delta t)$$

在两个采样时刻之间，系统的行为**不受 CBF 约束的直接保护**。

### 2.3 采样间隔内的安全穿越

```
时间轴:
  kΔt                    (k+1)Δt
   |                          |
   |  ←  这段区间内发生了什么？→  |
   |                          |
   b(s(kΔt)) ≥ 0 ✓       b(s((k+1)Δt)) ≥ 0 ✓
   
   但中间可能出现:
   b(s(t)) < 0,  ∃ t ∈ (kΔt, (k+1)Δt)
   
   ★ 采样点处都"安全"，但中间穿越了不安全区域
```

**直观比喻**：每隔 1 秒拍一张照片确认车辆没有越线，但两张照片之间车辆可能已经越线又回到了车道内——照片上看到的都是"安全"的，但实际已经发生了危险。

### 2.4 数学本质

连续时间 CBF 条件 $\dot{b}(s) + \alpha(b(s)) \geq 0$ 的离散化版本为：

$$\frac{b(s_{k+1}) - b(s_k)}{\Delta t} + \alpha(b(s_k)) \geq 0$$

这只是一个**一阶近似**。当 $\Delta t$ 较大或系统动态较快时，高阶项不可忽略：

$$b(s_{k+1}) = b(s_k) + \dot{b}(s_k)\Delta t + \frac{1}{2}\ddot{b}(s_k)\Delta t^2 + \mathcal{O}(\Delta t^3)$$

离散化误差 $\frac{1}{2}\ddot{b}(s_k)\Delta t^2$ 在安全边界附近（$b(s_k) \approx 0$）可能导致 $b(s_{k+1}) < 0$。

### 2.5 影响程度

- $\Delta t$ 越小 → 离散化误差越小 → 安全性越高（但计算代价越大）
- 系统动态越快（$\ddot{b}$ 越大） → 离散化误差越大 → 穿越风险越高
- 在安全边界附近 → 即使很小的误差也可能导致不安全

## 3. 局限二：动力学模型的不确定性

### 3.1 CBF 约束对动力学的依赖

QP 中的约束矩阵 $G, h$ 通过李导数计算，而李导数直接依赖动力学模型：

$$L_f b(s) = \nabla b(s) \cdot f(s), \quad L_g b(s) = \nabla b(s) \cdot g(s)$$

$$L_f^2 b(s) = \nabla (L_f b) \cdot f(s), \quad L_g L_f b(s) = \nabla (L_f b) \cdot g(s)$$

**每一个李导数的计算都需要精确知道 $f(s)$ 和 $g(s)$。**

### 3.2 模型误差的传播

设真实动力学为 $f_{\text{true}}, g_{\text{true}}$，模型为 $f_{\text{model}}, g_{\text{model}}$，误差为：

$$\delta f = f_{\text{true}} - f_{\text{model}}, \quad \delta g = g_{\text{true}} - g_{\text{model}}$$

则 QP 使用的约束与真实约束之间的偏差为：

$$\delta G = -\nabla(L_f b) \cdot \delta g, \quad \delta h = \nabla(L_f b) \cdot \delta f + \cdots$$

**关键问题**：QP 求解的是 $G_{\text{model}} u \leq h_{\text{model}}$，但真正需要满足的是 $G_{\text{true}} u \leq h_{\text{true}}$。

```
模型认为的安全约束:   G_model · u ≤ h_model  →  解出 u*
真实的安全约束:       G_true · u ≤ h_true    →  u* 可能不满足!

     u 空间
     │
     │  ╱ G_model·u = h_model  (模型约束边界)
     │ ╱
     │╱  ← u* 在这里（模型认为可行）
     │╲
     │ ╲  G_true·u = h_true  (真实约束边界)
     │  ╲
     │   ╲ ← u* 在这里实际上不可行!
     └────────────────────►
```

### 3.3 模型误差的来源

| 误差来源   | 说明              | 影响            |
| ------ | --------------- | ------------- |
| 参数不确定性 | 质量、摩擦系数等物理参数不精确 | $f, g$ 的系数有偏差 |
| 未建模动态  | 空气动力学、轮胎非线性等被忽略 | $f, g$ 的结构不完整 |
| 外部扰动   | 风、路面不平、传感器噪声    | $f, g$ 的瞬时值偏离 |
| 线性化误差  | 非线性系统在工作点附近线性化  | $f, g$ 仅在局部准确 |
| 神经网络近似 | 用 NN 学习动力学模型    | 训练覆盖外的区域误差大   |

### 3.4 误差累积效应

即使单步误差很小，在多步执行中误差会累积：

$$b(s_{k+1}) = b(s_k) + \underbrace{(L_f b + L_g b \cdot u_k)}_{\text{模型预测}} \Delta t + \underbrace{\epsilon_k}_{\text{模型误差}}$$

经过 $N$ 步后：

$$b(s_N) = b(s_0) + \sum_{k=0}^{N-1} (\cdots) \Delta t + \sum_{k=0}^{N-1} \epsilon_k$$

误差累积项 $\sum \epsilon_k$ 可能使 $b(s_N)$ 逐渐偏离预期，最终穿越安全边界。

## 4. 局限三：QP 无可行解——控制能力的物理极限

### 4.1 可行域为空的情况

QP 的可行域是 CBF 约束与控制输入边界的交集：

$$\mathcal{U}_{\text{feasible}} = \{u: G u \leq h\} \cap \{u: u_{\min} \leq u \leq u_{\max}\}$$

当这个交集为空时，QP 无解。

**典型场景**（以 CARLA 紧急制动为例）：

$$\text{CBF 要求:} \quad a \leq -8 \, \text{m/s}^2 \quad \text{（需要大力刹车）}$$

$$\text{物理极限:} \quad a \geq -6 \, \text{m/s}^2 \quad \text{（刹车最大只能提供 6 m/s²）}$$

$$\Rightarrow \quad \mathcal{U}_{\text{feasible}} = \emptyset$$

### 4.2 无解时的退化处理

当 QP 无可行解时，通常的处理方式都**放弃了严格的安全保证**：

**方式一：松弛约束**

引入松弛变量 $\delta \geq 0$，将硬约束变为软约束：

$$\min_{u, \delta} \quad \frac{1}{2}\|u\|^2 + q^T u + \lambda \delta^2$$

$$\text{s.t.} \quad G u \leq h + \delta, \quad u_{\min} \leq u \leq u_{\max}$$

→ 安全约束被允许违反（$\delta > 0$），系统可能进入不安全状态。

**方式二：回退到上一时刻控制**

$$u_k = u_{k-1}$$

→ 上一时刻的控制不一定适合当前状态，可能不安全。

**方式三：仅满足输入边界**

$$u_k = \text{clip}(q_k, u_{\min}, u_{\max})$$

→ 完全忽略 CBF 约束，安全性无任何保证。

### 4.3 不可恢复状态

存在某些状态，从这些状态出发，即使施加最大控制力也无法避免不安全：

$$\forall u \in [u_{\min}, u_{\max}], \quad b(s_{k+1}) < 0$$
	
这些状态被称为**不可恢复状态**（non-recoverable states）。一旦系统进入不可恢复状态，CBF-QP 完全无能为力。

```
状态空间:

  安全集 C: b(s) ≥ 0
  ┌─────────────────────────────────┐
  │                                  │
  │   可控安全区域                    │
  │   (CBF-QP 有效)                 │
  │   ┌─────────────────┐           │
  │   │                 │           │
  │   │   ● 当前状态     │           │
  │   │                 │           │
  │   └─────────────────┘           │
  │         ↓ 接近边界               │
  │   ┌─────────────────┐           │
  │   │  不可恢复区域    │           │
  │   │  (CBF-QP 无效!) │           │
  │   │  QP 无可行解     │           │
  │   └─────────────────┘           │
  │                                  │
  └─────────────────────────────────┘
  不安全集: b(s) < 0
```

## 5. 局限四：障碍函数定义的固有缺陷

### 5.1 安全集的数学表示

CBF 的安全性完全建立在障碍函数 $b(s)$ 对安全集的正确刻画上：

$$\mathcal{C} = \{s \in \mathbb{R}^n : b(s) \geq 0\}$$

**理想情况**：$\mathcal{C}$ 精确等于真实安全集 $\mathcal{C}_{\text{true}}$。

**现实情况**：$\mathcal{C}$ 与 $\mathcal{C}_{\text{true}}$ 之间存在偏差。

### 5.2 偏差的两种方向

**方向一：过于保守**（$\mathcal{C} \subset \mathcal{C}_{\text{true}}$）

$$b(s) \geq 0 \implies s \text{ 真的安全} \quad \text{（安全保证有效）}$$

$$\text{但} \quad s \text{ 真的安全} \not\implies b(s) \geq 0 \quad \text{（部分安全状态被排除）}$$

→ 系统过于保守，可能丧失任务性能（如不必要的紧急制动）。

**方向二：过于宽松**（$\mathcal{C} \supset \mathcal{C}_{\text{true}}$）

$$b(s) \geq 0 \not\implies s \text{ 真的安全} \quad \text{（安全保证失效！）}$$

→ 系统"以为"安全，但实际上不安全。**这是最危险的情况。**

```
过于保守:                        过于宽松:
┌────────────────┐              ┌────────────────┐
│ 真实安全集      │              │ b(s)≥0 定义的集 │
│ ┌──────────┐   │              │ ┌──────────┐   │
│ │ b(s)≥0   │   │              │ │          │   │
│ │ (更小)   │   │              │ │  b(s)≥0  │   │
│ └──────────┘   │              │ │  (更大)  │   │
│                │              │ │ ┌──────┐ │   │
└────────────────┘              │ │ │真实  │ │   │
                                │ │ │安全集│ │   │
                                │ │ └──────┘ │   │
                                │ │  ↑ 不安全│   │
                                │ │    但被认为安全│
                                │ └──────────┘   │
                                └────────────────┘
```

### 5.3 复杂安全集的近似困难

许多实际场景的安全集是非凸的、不连通的、或高维的：

- **非凸安全集**：如车辆需要同时避开多个障碍物，安全集是多个圆的补集的交集
- **高维安全集**：状态空间维度高时，$b(s)$ 难以精确刻画安全边界
- **动态安全集**：安全集本身随时间变化（如移动障碍物）

这些情况下，通常只能用简单的 $b(s)$（如二次函数）来近似，近似误差不可避免。

### 5.4 相对阶问题

对于相对阶 $r > 1$ 的系统（控制 $u$ 不直接出现在 $\dot{b}$ 中），需要使用 HOCBF：

$$b(s) \geq 0 \implies \psi_1(s) \geq 0 \implies \cdots \implies \psi_{r-1}(s) \geq 0$$

每一层引入一个 class-K 函数 $\alpha_i$，层层传递后：

- 约束条件的保守性逐层放大
- 对 $b(s)$ 的高阶导数计算要求更高精度的动力学模型
- 可行的 $b(s)$ 形式更加受限

## 6. 局限五：CBF 参数的 NN 学习不可靠性

### 6.1 参数的作用

CBF 约束中的 class-K 函数参数 $p$ 控制着安全约束的"激进程度"：

$$L_g b(s) \cdot u + L_f b(s) + p \cdot b(s) \geq 0$$

- $p$ 较大 → 约束更紧 → 更早开始避障/制动 → 更安全但更保守
- $p$ 较小 → 约束更松 → 更晚开始避障/制动 → 更灵活但更危险

### 6.2 NN 学习参数的风险

BarrierNet 用神经网络学习自适应参数：

$$p_t = 4 \cdot \sigma(\text{NN}_\phi(o_t))$$

但 NN 的学习存在以下风险：

**1. 分布外泛化失败**

NN 在训练分布内可能学到合理的 $p$，但在 OOD 输入下可能输出极端值：

```
训练分布内:  o_t → NN → p_t = 2.0  (合理)
OOD 输入:    o_t → NN → p_t = 0.01 (太小，约束过松)
             o_t → NN → p_t = 3.99 (太大，QP 可能无解)
```

**2. 参数的对抗性**

训练过程中，网络可能学到"作弊"策略——输出使 CBF 损失很小但实际上不安全的参数：

- 输出极小的 $p$ → $\mathcal{L}_{\text{CBF}}$ 容易满足（约束很松），但安全性降低
- 输出极大的 $p$ → 约束很紧，但 QP 经常无可行解，回退到不安全的控制

**3. 缺乏参数的理论最优性保证**

NN 学到的 $p_t$ 没有理论保证说明它是最优的或甚至合理的。参数的"好坏"只能通过实验来评估，而非数学证明。

### 6.3 固定参数 vs 自适应参数的权衡

| | 固定参数（手工设定） | 自适应参数（NN 学习） |
|---|---|---|
| 优点 | 可分析、可验证 | 可适应不同状态 |
| 缺点 | 所有状态用同一保守度 | OOD 下可能输出不合理值 |
| 安全性 | 保守但可预测 | 灵活但不可预测 |

## 7. 局限六：数值精度与求解器误差

### 7.1 浮点运算的有限精度

QP 求解器使用有限精度浮点运算（通常为 float32 或 float64）：

- 约束满足的判断存在容差：$Gu \leq h + \epsilon_{\text{tol}}$（通常 $\epsilon_{\text{tol}} \sim 10^{-6}$）
- 这意味着 $u^*$ 可能**略微违反** CBF 约束

### 7.2 安全边界处的数值敏感性

当系统接近安全边界（$b(s) \approx 0$）时：

- 约束矩阵 $G$ 和 $h$ 中的各项接近零，数值条件数恶化
- KKT 系统的 Jacobian 接近奇异，隐式微分的梯度可能不准确
- QP 求解器的收敛速度下降，可能提前终止于不精确的解

```
远离安全边界:  b(s) = 10.0  →  数值稳定  →  QP 解精确
接近安全边界:  b(s) = 0.001 →  数值敏感  →  QP 解可能有误差
在安全边界上:  b(s) = 0.000 →  病态条件  →  QP 解不可靠
```

### 7.3 梯度反传的数值问题

训练时，梯度通过 QP 层的隐式微分反传：

$$\frac{\partial u^*}{\partial \theta} = -\left(\frac{\partial \text{KKT}}{\partial u^*}\right)^{-1} \frac{\partial \text{KKT}}{\partial \theta}$$

当 KKT 系统的 Jacobian $\frac{\partial \text{KKT}}{\partial u^*}$ 接近奇异时：

- 逆矩阵的数值计算不稳定
- 梯度可能爆炸或消失
- 训练过程可能发散

### 7.4 累积效应

虽然单次 QP 求解的数值误差很小（$\sim 10^{-6}$），但在长时间运行中：

- 误差可能累积（特别是在安全边界附近反复震荡时）
- 某些时间步的误差可能使 $b(s)$ 从正值变为负值
- 一旦 $b(s) < 0$，前向不变性保证完全失效

## 8. 局限七：多约束冲突与优先级困境

### 8.1 多约束场景

实际系统中通常存在多个安全约束：

$$G = \begin{bmatrix} G_1 \\ G_2 \\ \vdots \\ G_m \end{bmatrix}, \quad h = \begin{bmatrix} h_1 \\ h_2 \\ \vdots \\ h_m \end{bmatrix}$$

例如：
- $G_1, h_1$：避障约束
- $G_2, h_2$：车道保持约束
- $G_3, h_3$：速度限制约束
- $G_4, h_4$：加速度限制约束

### 8.2 约束冲突

当多个约束同时收紧时，可行域可能变为空集：

$$\{u: G_1 u \leq h_1\} \cap \{u: G_2 u \leq h_2\} \cap [u_{\min}, u_{\max}] = \emptyset$$

**典型场景**：车辆同时面临"前方障碍物需要左转"和"左侧车道有车不能变道"两个约束，两个约束无法同时满足。

### 8.3 优先级困境

当约束冲突时，需要决定优先满足哪个约束。但：

- CBF 理论本身**不提供约束优先级的机制**
- 简单的松弛方法（如加权松弛）无法保证最重要的约束一定被满足
- 错误的优先级选择可能导致更严重的安全问题

```
场景: 前方有行人 + 左侧有卡车

  约束1 (避行人): 必须右转或刹车
  约束2 (避卡车): 不能左转
  
  如果同时要求右转且刹车，但刹车距离不够...
  → 必须选择: 优先避行人（右转）还是优先避卡车（不左转）?
  → CBF-QP 无法自动做出正确选择
```

## 9. 局限八：感知噪声与状态估计误差

### 9.1 CBF 假设完全状态可观测

标准 CBF-QP 框架假设当前状态 $s_t$ 是精确已知的：

$$G(s_t, p_t) u \leq h(s_t, p_t)$$

但在使用视觉等感知模态时，状态是通过估计得到的：

$$\hat{s}_t = \text{Estimator}(\text{sensors})$$

估计误差 $\delta s = s_t - \hat{s}_t$ 导致约束基于**错误的状态**构造。

### 9.2 估计误差的安全影响

设真实状态为 $s_t$，估计状态为 $\hat{s}_t = s_t + \delta s$，则：

$$G(\hat{s}_t) u \leq h(\hat{s}_t) \quad \text{（QP 使用的约束）}$$

$$G(s_t) u \leq h(s_t) \quad \text{（真正需要满足的约束）}$$

两者之间的差异取决于 $\delta s$ 和李导数对状态的敏感度：

$$\Delta G \approx \frac{\partial G}{\partial s} \delta s, \quad \Delta h \approx \frac{\partial h}{\partial s} \delta s$$

**在安全边界附近**，即使很小的 $\delta s$ 也可能导致约束的质变——QP 认为可行的解在真实约束下可能不可行。

### 9.3 BarrierNet 的特殊问题

BarrierNet 的 CBF 参数 $p_t$ 也依赖观测输入：

$$p_t = 4 \cdot \sigma(\text{NN}(o_t))$$

感知噪声不仅影响约束矩阵 $G, h$，还影响 CBF 参数 $p_t$，形成**双重误差叠加**：

$$G(\hat{s}_t, \hat{p}_t) u \leq h(\hat{s}_t, \hat{p}_t)$$

两个输入都有误差，使得约束与真实安全需求之间的偏差更大。

## 10. 所有局限的系统性总结

### 10.1 局限性分类

| 类别 | 局限 | 根本原因 | 严重程度 | 可缓解性 |
|------|------|---------|---------|---------|
| **理论层面** | 连续-离散裂缝 | CBF 理论是连续的，执行是离散的 | ⭐⭐⭐ | 减小 $\Delta t$，但无法消除 |
| **理论层面** | 前向不变性仅为充分条件 | $b(s) \geq 0$ 不等于"真正安全" | ⭐⭐⭐⭐ | 需要更精确的 $b(s)$ 定义 |
| **模型层面** | 动力学模型误差 | $f, g$ 不精确 | ⭐⭐⭐⭐ | 鲁棒 CBF 可部分缓解 |
| **模型层面** | 障碍函数定义不当 | $b(s)$ 与真实安全集不一致 | ⭐⭐⭐ | 数据驱动学习 $b(s)$ |
| **求解层面** | QP 无可行解 | 控制能力 < 安全需求 | ⭐⭐⭐⭐⭐ | 松弛约束（牺牲安全性） |
| **求解层面** | 数值精度有限 | 浮点运算 | ⭐⭐ | 使用 double 精度 |
| **学习层面** | NN 参数不可靠 | OOD 泛化失败 | ⭐⭐⭐ | 更充分的训练 |
| **系统层面** | 多约束冲突 | 物理矛盾 | ⭐⭐⭐ | 约束优先级设计 |
| **系统层面** | 感知噪声 | 状态估计不精确 | ⭐⭐⭐ | 鲁棒 CBF |

### 10.2 局限性的相互作用

这些局限不是孤立的，它们之间存在**放大效应**：

```
模型误差 ──► G, h 不准确 ──► QP 解不满足真实约束
    │                              │
    │                              ▼
    │                    安全边界估计错误
    │                              │
    ▼                              ▼
感知噪声 ──► 状态估计错误 ──► 在"假安全边界"处运行
    │                              │
    │                              ▼
    └──► NN 参数 p 不合理 ──► 约束松紧不当
                                   │
                                   ▼
                         离散化误差 ──► 穿越安全边界
                                   │
                                   ▼
                          系统进入不安全状态
```

**最坏情况**：多个局限同时作用——模型有误差 + 感知有噪声 + 接近安全边界 + 离散时间步长大——层层叠加，安全保证完全失效。

## 11. SafePVC 的 SBC 如何补充这些局限

### 11.1 SBC 与 CBF 的本质区别

| 维度 | CBF-QP（BarrierNet） | SBC（SafePVC） |
|------|---------------------|----------------|
| **保障类型** | 确定性前向不变性 | 概率安全性 $\mathbb{P}[\text{Safe}] \geq p$ |
| **时间域** | 连续时间（离散执行有缝隙） | 离散时间（天然匹配） |
| **对扰动的处理** | 不考虑（假设确定性系统） | 显式建模 $\Delta s \sim \mu$ |
| **对模型的依赖** | 需要精确的 $f, g$ | 通过数据驱动估计，容忍模型误差 |
| **验证方式** | 无离线验证 | $\alpha$-$\beta$-CROWN 形式化验证 |

### 11.2 逐条补充

| CBF-QP 的局限 | SBC 如何补充 |
|-------------|------------|
| 连续-离散裂缝 | SBC 的期望递减条件在**离散时间**下建立，天然覆盖采样间隔 |
| 动力学模型误差 | SBC 显式建模扰动分布 $\Delta s \sim \mu$，将模型误差纳入扰动中 |
| QP 无可行解 | SBC 不依赖 QP 求解，即使 QP 失效，概率安全界仍然有效 |
| 障碍函数 $b(s)$ 定义不当 | SBC 使用独立的随机障碍证书 $B(s)$，不依赖 $b(s)$ 的正确性 |
| NN 参数不可靠 | SBC 的 CEGIS 反例机制可以暴露不合理参数并驱动修正 |
| 数值精度有限 | SBC 的安全界 $\beta$ 可以吸收数值误差（通过保守的区间传播） |
| 多约束冲突 | SBC 从全局概率视角评估安全性，不受单约束冲突影响 |
| 感知噪声 | SafePVC 的感知模型（cGAN + 蒸馏 MLP）带有 Lipschitz 约束，显式处理感知误差 |

### 11.3 互补关系的核心

**CBF-QP** 像一个**有缝隙的栅栏**：
- 大部分时候能挡住危险
- 但在缝隙处（离散间隔、模型误差、QP 无解等）可能漏过去

**SBC** 像一张**覆盖全局的安全网**：
- 即使栅栏有缝隙漏过去了，安全网仍然提供概率层面的兜底
- 安全网不是完美的（概率下界 $p < 1$），但它覆盖了栅栏的所有缝隙

**两者结合** = 栅栏 + 安全网 = **双重安全机制**

## 12. 结论：双重安全机制的必要性

### 12.1 单一机制的不足

- **仅用 BarrierNet（CBF-QP）**：运行时安全约束强，但存在上述 8 个根本性局限，无法保证 100% 安全，且缺乏离线验证来量化安全水平
- **仅用 SafePVC（SBC）**：有离线概率安全验证，但运行时没有显式安全约束，面对 OOD 输入时脆弱

### 12.2 双重机制的优势

$$\text{SafePVC + QP} = \underbrace{\text{CBF-QP}}_{\text{运行时确定性保护}} + \underbrace{\text{SBC}}_{\text{离线概率安全验证}}$$

- **CBF-QP** 负责"治标"——在每一步防止不安全动作被执行
- **SBC** 负责"治本"——从全局概率角度验证系统长期安全性
- CBF-QP 的局限性被 SBC 覆盖，SBC 的保守性被 CBF-QP 缓解

### 12.3 最终安全保证

在 SafePVC + QP 框架下，系统的安全保证为：

$$\mathbb{P}\left[\forall t \geq 0: s_t \in \mathcal{C}\right] \geq p_{\text{SBC}}$$

其中 $p_{\text{SBC}}$ 是经过双重安全机制后的概率安全下界，**预期高于单独使用 SBC 时的下界**（因为 CBF-QP 将系统约束在安全集内，降低了 SBC 验证的保守性）。

虽然 $p_{\text{SBC}} < 1$（无法达到 100%），但双重机制使得 $p_{\text{SBC}}$ 更接近 1，同时将运行时的不安全行为概率降至最低。

> **核心启示**：在安全关键的控制系统中，没有任何单一机制能够提供 100% 的安全保证。真正的安全性来自于**多层互补的安全机制**——每一层覆盖其他层的漏洞，层层叠加，将风险降至可接受的水平。这正是 SafePVC + QP 框架的设计理念。
