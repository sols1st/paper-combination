## 第3页：研究背景——为什么需要安全学习？

**核心问题：** 单次失败 = 灾难性后果

**现实场景举例：**
- 手术机器人：0.1% 的失误可能致命
- 自动驾驶：一次错误转向 = 严重事故
- 工业协作机器人：任何失控 = 人身伤害

**当前大型模型的困境：**
- 大模型（LLM驱动的规划、大规模视觉模型）在复杂任务上表现出色
- **但它们不可信——没有安全保证**

**结论：** 安全学习是 AI 驱动机器人的核心需求

---

## 第4页：现有安全方法的局限性

**基于障碍函数的方法（Barrier-based Methods）** 是主流安全学习方法

**三大致命缺陷：**

| 缺陷 | 说明 | 后果 |
|------|------|------|
| **不可扩展** | 只能在神经网络最后一层放单个安全层 | 无法处理复杂任务 |
| **难以训练** | 优化景观复杂，参数敏感 | 复杂任务上无法收敛 |
| **输出不稳定** | 噪声输入下输出剧烈波动 | 无法部署到真实机器人 |

**现有合并方法也不行：**
- 混合专家模型（MoE）：可扩展但丢失安全保证
- 预测不确定性量化：能处理噪声但丢失安全保证
- 模型合并（LoRA等）：能合并但丢失安全保证

## 第5页：核心思想——ABNet

**灵感来源：** Transformer 的多头注意力机制

**关键设计：**

```
           输入观测
              │
     ┌────────┼────────┐
     ▼        ▼        ▼
  [头1]    [头2]    [头h]     ← 每个头关注不同特征
     │        │        │
     ▼        ▼        ▼
    u₁       u₂       uₕ
     │        │        │
     └────────┼────────┘
              ▼
     u = w₁u₁ + w₂u₂ + ... + wₕuₕ   ← 加权凸组合
              │
              ▼
       最终安全控制输出
```

**三大优势：**
- **可扩展：** 增量添加新头，无需重新训练整个模型
- **鲁棒：** 多头融合天然过滤噪声
- **可证明安全：** 有严格数学定理保证

## 第7页：预备知识——仿射控制系统

**系统模型（公式 1）：**

$$\dot{\boldsymbol{x}} = f(\boldsymbol{x}) + g(\boldsymbol{x})\boldsymbol{u}$$

**各符号含义：**

| 符号 | 含义 | 例子（自动驾驶） |
|------|------|-----------------|
| $\boldsymbol{x} \in \mathbb{R}^n$ | 系统状态 | [位置x, 位置y, 朝向角θ, 速度v] |
| $\dot{\boldsymbol{x}}$ | 状态变化率 | 速度、加速度等 |
| $f(\boldsymbol{x})$ | 自然动力学（无控制时） | 惯性滑行 |
| $g(\boldsymbol{x})$ | 控制影响矩阵 | 方向盘/油门如何改变运动 |
| $\boldsymbol{u} \in U$ | 控制输入 | [转向率, 加速度] |
| $U$ | 控制约束集 | 最大转角、最大加速度 |

**"仿射"的含义：** 对控制 $\boldsymbol{u}$ 是线性的，但对状态 $\boldsymbol{x}$ 可以非线性——这个结构对后续 QP 优化至关重要

---

## 第8页：预备知识——前向不变性

**定义 2.1（前向不变性）：**

集合 $C \subset \mathbb{R}^n$ 对系统前向不变，如果从任意 $\boldsymbol{x}(0) \in C$ 出发，在合适的控制 $\boldsymbol{u}$ 下：

$$\boldsymbol{x}(t) \in C, \quad \forall t \geq 0$$

**直观理解：**

```
  ┌─────────────────────┐
  │   安全区域 C         │
  │                     │
  │   ● → → → → ●      │  轨迹始终在 C 内
  │  (起点)     (终点)   │
  │                     │
  └─────────────────────┘
      ✗ 危险区域 ✗
```

**例子：** C = {机器人离障碍物距离 ≥ R}，前向不变 = 永远不撞

---

## 第9页：预备知识——K 类函数与相对度

**定义 2.2（K 类函数）：**

$\alpha: [0, a) \to [0, \infty)$，严格递增，$\alpha(0) = 0$

- 例子：$\alpha(r) = r$，$\alpha(r) = r^2$，$\alpha(r) = 1 - e^{-r}$
- 作用：调节"离安全边界越近，约束越强"的程度

**定义 2.3（相对度）：**

需要对 $b$ 微分多少次，控制 $\boldsymbol{u}$ 才显式出现在导数中

**相对度 m=1：** $\dot{b}$ 中直接包含 $\boldsymbol{u}$ → 控制直接影响安全约束

**相对度 m=2：** $\dot{b}$ 中不含 $\boldsymbol{u}$，但 $\ddot{b}$ 中包含 → 控制间接影响安全约束

**意义：** 相对度 > 1 时，标准 CBF 无法使用 → 需要 HOCBF

---

## 第10页：预备知识——HOCBF 递归构造

**问题：** 当相对度 $m > 1$ 时，控制 $\boldsymbol{u}$ 不直接出现在安全约束的一阶导数中

**解决：递归构造中间约束（公式 2）：**

$$\psi_i(\boldsymbol{x}) := \dot{\psi}_{i-1}(\boldsymbol{x}) + \alpha_i(\psi_{i-1}(\boldsymbol{x})), \quad i \in \{1, \ldots, m\}$$

其中 $\psi_0(\boldsymbol{x}) := b(\boldsymbol{x})$

**以 m=2 为例：**

```
第0层: ψ₀(x) = b(x)         原始安全约束（如：距离-R）
第1层: ψ₁(x) = ψ̇₀(x) + α₁(ψ₀(x))   中间约束
第2层: ψ₂(x) = ψ̇₁(x) + α₂(ψ₁(x))   最终约束（u出现在这里！）
```

**倒推法思想：**
- 保证 ψ₂ ≥ 0 → 可保证 ψ₁ ≥ 0 → 可保证 ψ₀ = b(x) ≥ 0
- 在第 m 层，控制 $\boldsymbol{u}$ 出现，可以通过选择 $\boldsymbol{u}$ 来保证安全

---

## 第11页：预备知识——安全集与 HOCBF 定义

**安全集序列（公式 3）：**

$$C_i := \{\boldsymbol{x} \in \mathbb{R}^n : \psi_{i-1}(\boldsymbol{x}) \geq 0\}, \quad i \in \{1, \ldots, m\}$$

系统必须同时满足所有安全集：$\boldsymbol{x} \in \cap_{i=1}^m C_i$

**定义 2.4（HOCBF）：** 函数 $b$ 是 HOCBF，如果存在 K 类函数使得：

$$\sup_{\boldsymbol{u} \in U} [L_f \psi_{m-1}(\boldsymbol{x}) + [L_g \psi_{m-1}(\boldsymbol{x})]\boldsymbol{u} + \alpha_m(\psi_{m-1}(\boldsymbol{x}))] \geq 0 \tag{4}$$

**逐项解释：**

| 项 | 物理意义 |
|----|----------|
| $L_f \psi_{m-1}$ | 无控制时安全约束的变化率 |
| $[L_g \psi_{m-1}]\boldsymbol{u}$ | 控制对安全约束变化率的影响 |
| $\alpha_m(\psi_{m-1})$ | 安全裕度（离边界越近，要求越严格） |

**含义：** 存在某个控制 $\boldsymbol{u}$ 使系统保持安全

---

## 第12页：预备知识——HOCBF 安全定理

**定理 2.5：** 如果 $\boldsymbol{x}(0) \in \cap_{i=1}^m C_i$，则任何满足 HOCBF 约束 (4) 的 Lipschitz 连续控制器 $\boldsymbol{u}(t)$ 使得 $\cap_{i=1}^m C_i$ 前向不变

**即：** $b_j(\boldsymbol{x}(t)) \geq 0, \forall t \geq 0$

**为什么重要：**
- HOCBF 约束 (4) 对 $\boldsymbol{u}$ 是**线性的**
- 可以嵌入到二次规划（QP）中高效求解
- QP 可以在毫秒级求解 → 适合实时控制

**CBF 是 HOCBF 的特殊情况：** 令 $m=1$ 即可得到标准 CBF

---

## 第13页：预备知识——BarrierNet

**BarrierNet 是什么？**

将 CBF/HOCBF 约束转化为神经网络的**可微层**

```
传统方法:
  手动选择 CBF 参数 → 固定参数 → 可能不是最优

BarrierNet:
  CBF 参数变成可学习参数 → 用数据自动优化 → 更好的性能
```

**核心技术：可微 QP（dQP）**
- OptNet [3] 使 QP 求解过程可微
- 梯度可以通过 QP 层反向传播
- CBF 参数（如 K 类函数参数）可被数据优化

**传统 BarrierNet 的局限：**
- 只有单头（h=1）
- 作为最后一层使用
- 不可扩展

## 第14页：问题表述

**给定：**

| 条件 | 含义 | 例子 |
|------|------|------|
| (a) 系统动力学 | 公式 (1) 的仿射系统 | 自行车模型 |
| (b) 标称控制器 | $\boldsymbol{\pi}^*(\boldsymbol{x}) = \boldsymbol{u}^*$，提供训练标签 | MPC 控制器 |
| (c) 安全约束 | $b_j(\boldsymbol{x}) \geq 0, j \in S$ | 与障碍物距离 ≥ R |
| (d) 神经网络控制器 | $\pi(\boldsymbol{x}, \boldsymbol{z}|\theta) = \boldsymbol{u}$ | CNN + BarrierNet |

**优化目标（公式 5）：**

$$\theta^* = \arg\min_\theta \mathbb{E}_{\boldsymbol{x}, \boldsymbol{z}} [\ell(\boldsymbol{\pi}^*(\boldsymbol{x}), \pi(\boldsymbol{x}, \boldsymbol{z}|\theta))]$$

**约束：** 满足所有安全约束 (c) 和动力学约束 (a)

**本质：** 带安全约束的模仿学习——既要学得好，还要学得安全

## 第15页：ABNet 架构——多头 BarrierNet

**每个头是一个可微 QP（公式 6）：**

$$\boldsymbol{u}_k = \arg\min_{\boldsymbol{u}(t) \in U} \frac{1}{2}\boldsymbol{u}^T H(\boldsymbol{z}_k|\boldsymbol{\theta}_{h,k})\boldsymbol{u} + F^T(\boldsymbol{z}_k|\boldsymbol{\theta}_{f,k})\boldsymbol{u}$$

**目标函数解释：**

| 项 | 含义 |
|----|------|
| $\frac{1}{2}\boldsymbol{u}^T H \boldsymbol{u}$ | 控制能量惩罚（不希望控制太大） |
| $F^T \boldsymbol{u}$ | 引导控制方向 |
| $H^{-1}F$ | "参考控制"——不考虑安全时模型的期望输出 |

**$H$ 正定** → QP 有唯一最优解

**忽略安全约束时：** 最优解为 $\boldsymbol{u} = -H^{-1}F$（参考控制）

---

## 第16页：ABNet 架构——HOCBF 安全约束

**每个头必须满足的约束（公式 7）：**

$$L_f \psi_{j,m-1} + [L_g \psi_{j,m-1}]\boldsymbol{u} + p_{m,k}\alpha_{j,m}(\psi_{j,m-1}) \geq 0 \tag{7a}$$

$$\psi_{j,i} = \dot{\psi}_{j,i-1} + p_i \alpha_{j,i}(\psi_{j,i-1}), \quad i \in \{1, \ldots, m-1\} \tag{7b}$$

$$\psi_{j,0} = b_j(\boldsymbol{x}) \tag{7c}$$

**关键参数：**

| 参数 | 作用 | 特性 |
|------|------|------|
| $p_{m,k} \geq 0$ | 最顶层惩罚，控制保守程度 | **每个头不同**（带下标 k） |
| $p_i \geq 0$ | 中间层惩罚 | **所有头共享**（不带 k） |
| $\alpha_{j,i}$ | K 类函数 | 可学习 |

**$p_{m,k}$ 的物理意义：**
- 值越大 → 系统越保守 → 离安全边界越远 → 但任务性能可能下降
- 每个头有不同的保守程度 → 融合后取得平衡

---

## 第17页：ABNet 架构——参数结构

**参数层次：**

```
ABNet 所有可训练参数 θ:
├── 头 1: (θ_h,1, θ_f,1, θ_p,1^m)    ← 独立参数
├── 头 2: (θ_h,2, θ_f,2, θ_p,2^m)    ← 独立参数
├── ...
├── 头 h: (θ_h,h, θ_f,h, θ_p,h^m)    ← 独立参数
└── 共享参数: θ_p = (θ_p^1, ..., θ_p^{m-1})  ← 交叉连接
```

**独立 vs 共享：**
- **独立（每头不同）：** $H, F, p_{m,k}$ → 各头有不同的控制目标和保守度
- **共享（所有头相同）：** $p_i$ → 确保 HOCBF 中间构造一致

**观测灵活性：**
- 可以共享观测：$\boldsymbol{z}_k = \boldsymbol{z}, \forall k$
- 也可以用不同观测：$\boldsymbol{z}_k$ 来自不同传感器子集

## 第18页：注意力机制

**两种注意力模式：**

**模式 1：相同观测 + 不同参数**

```
输入 z ──→ [头1, p_m,1参数不同] ──→ u₁
      ├──→ [头2, p_m,2参数不同] ──→ u₂
      └──→ [头3, p_m,3参数不同] ──→ u₃
```

通过随机初始化，每个头学到不同的安全策略（类似集成学习）

**模式 2：不同观测 + 显式注意力**

```
z_左车道  ──→ [头1] ──→ u₁   关注左车道
z_右车道  ──→ [头2] ──→ u₂   关注右车道
z_前方    ──→ [头3] ──→ u₃   关注障碍物
```

每个头看到不同信息 → 合并后更全面

**多模态感知：** $\boldsymbol{z}_k$ 可来自不同感知模态（视觉、激光雷达、雷达等）

## 第19页：交叉连接

**设计：** 所有头共享中间层 HOCBF 构造参数 $\theta_p$

```
         共享参数 θ_p
            │
  ┌─────────┼─────────┐
  ▼         ▼         ▼
[头1的CBF] [头2的CBF] [头3的CBF]
 ψ₁,i      ψ₂,i      ψ₃,i      ← 中间层相同
  │         │         │
  ▼         ▼         ▼
 头1独有   头2独有    头3独有
 p_m,1     p_m,2     p_m,3      ← 保守度不同
```

**为什么需要共享中间层？**

1. **安全证明的需要：** 定理 3.1 的证明依赖于所有头的 $\psi_{j,i}$ 相同，才能提取公因子
2. **信息共享：** 所有头为同一物理系统生成控制，需要对"离安全边界多远"有一致理解

**类比：** 安全委员会——各委员从不同角度分析，但共享相同的基础评估框架

## 第20页：融合机制

**加权凸组合（公式 8）：**

$$\boldsymbol{u} = \sum_{k=1}^{h} w_k \boldsymbol{u}_k, \quad \sum_{k=1}^{h} w_k = 1, \quad w_k \geq 0$$

**为什么是凸组合？**

**原因 1——安全性保持：**
- HOCBF 约束对 $\boldsymbol{u}$ 是线性的
- 权重非负且和为1 → 融合后仍满足约束（后面证明）

**原因 2——物理合理：**
- 融合后的控制是各头的"加权平均"
- 不会产生超出各头控制范围的极端值

**原因 3——权重可学习：**
- $w_k$ 是可训练参数
- 网络自动学习哪个头更重要

**约束必要性：**
- $w_k \geq 0$：不允许负权重（否则可能破坏安全性）
- $\sum w_k = 1$：确保融合是"平均"而非"放大"

---

## 第21页：天然噪声滤波器

**ABNet 天然过滤噪声：**

假设每个头的输出 = 真实信号 + 噪声：

$$\boldsymbol{u}_k = \boldsymbol{u}_{\text{true}} + \boldsymbol{n}_k$$

各头的噪声不相关（关注不同特征），融合后：

$$\boldsymbol{u} = \underbrace{\boldsymbol{u}_{\text{true}} \sum w_k}_{= \boldsymbol{u}_{\text{true}}} + \underbrace{\sum w_k \boldsymbol{n}_k}_{\text{噪声减小}}$$

**当头数 h 足够大且 $w_k \approx 1/h$ 时：**
- 噪声标准差降低为原来的 $1/\sqrt{h}$
- 输出更平滑、更稳定

**类比：** 嘈杂房间里听取多人意见——综合多数人意见更接近真相

## 第22页：安全性定理 3.1

**定理 3.1（ABNet 的安全性）：**

**前提：** 系统初始安全 $b_j(\boldsymbol{x}(t_0)) \geq 0, \forall j \in S$

**结论：** ABNet 输出 $\boldsymbol{u}$ 保证系统永远安全

$$b_j(\boldsymbol{x}(t)) \geq 0, \quad \forall j \in S, \quad \forall t \geq t_0$$

**为什么重要？**

1. **多头融合不破坏安全性：** 直观上合并多个安全控制器可能不安全，但定理证明在 ABNet 设计下融合是安全的
2. **形式化保证 vs 经验保证：** 不是"测试中没出问题"，而是"数学上保证不出问题"

**初始不安全时：**
- HOCBF 的 Lyapunov 性质会将系统推向安全区域
- 可用不安全数据进行对抗训练

## 第23页：安全性证明——步骤 1-2

**证明思路总览：**

```
步骤1: 每个头满足 HOCBF 约束 (10)
  ↓
步骤2: 乘以权重 w_k (11)
  ↓
步骤3: 对所有头求和 (12)
  ↓
步骤4: 化简得到新 HOCBF 约束 (13)
  ↓
步骤5: 证明新约束是有效的 HOCBF
  ↓
步骤6: Nagumo 定理 → 前向不变性 (14)-(15)
  ↓
步骤7: 递归证明所有安全集 → b_j(x) ≥ 0
```

**步骤 1：** 每个头 $k$ 通过 QP 求解，满足：

$$L_f \psi_{j,m-1} + [L_g \psi_{j,m-1}]\boldsymbol{u}_k + p_{m,k}\alpha_{j,m}(\psi_{j,m-1}) \geq 0 \tag{10}$$

**步骤 2：** 乘以 $w_k \geq 0$（非负 → 不等式方向不变！）：

$$w_k L_f \psi_{j,m-1} + w_k[L_g \psi_{j,m-1}]\boldsymbol{u}_k + w_k p_{m,k}\alpha_{j,m}(\psi_{j,m-1}) \geq 0 \tag{11}$$

---

## 第24页：安全性证明——步骤 3-4

**步骤 3：** 对所有头求和（$h$ 个 $\geq 0$ 的不等式之和仍 $\geq 0$）：

$$\sum_{k=1}^{h} w_k L_f \psi_{j,m-1} + \sum_{k=1}^{h} w_k [L_g \psi_{j,m-1}]\boldsymbol{u}_k + \sum_{k=1}^{h} w_k p_{m,k}\alpha_{j,m}(\psi_{j,m-1}) \geq 0 \tag{12}$$

**步骤 4：关键化简**

由于 $L_f \psi_{j,m-1}$ 和 $L_g \psi_{j,m-1}$ **不依赖于 k**（共享中间层！），且 $\sum w_k = 1$：

$$L_f \psi_{j,m-1} + L_g \psi_{j,m-1}\underbrace{\left(\sum_{k=1}^{h} w_k \boldsymbol{u}_k\right)}_{\text{ABNet输出 } \boldsymbol{u}} + \underbrace{\sum_{k=1}^{h} w_k p_{m,k}\alpha_{j,m}(\psi_{j,m-1})}_{\text{新的K类函数}} \geq 0 \tag{13}$$

**这就是交叉连接的必要性！** 如果各头的 $\psi_{j,m-1}$ 不同，无法提取公因子，证明就无法完成。

---

## 第25页：安全性证明——步骤 5-6

**步骤 5：验证新约束是有效的 HOCBF**

- K 类函数的加权和仍是 K 类函数
- 公式 (13) 形式上与标准 HOCBF 一致
- 只是控制输入变成了 $\sum w_k \boldsymbol{u}_k$

**步骤 6：当 $\psi_{j,m-1} = 0$（安全边界上）时：**

$$\alpha_{j,m}(0) = 0 \text{（K类函数性质）} \Rightarrow \text{第三项消失}$$

$$L_f \psi_{j,m-1} + L_g \psi_{j,m-1} \cdot \boldsymbol{u} \geq 0 \tag{14}$$

利用 Lie 导数定义：

$$L_f \psi + L_g \psi \cdot \boldsymbol{u} = \frac{\partial \psi}{\partial \boldsymbol{x}}(f + g\boldsymbol{u}) = \frac{\partial \psi}{\partial \boldsymbol{x}}\dot{\boldsymbol{x}} = \dot{\psi}_{j,m-1} \geq 0 \tag{15}$$

**$\dot{\psi}_{j,m-1} \geq 0$ 的含义：** 在安全边界上，安全约束的变化率非负 → 系统不会穿越到不安全侧

---

## 第26页：安全性证明——步骤 7 与 Nagumo 定理

**Nagumo 定理（集合不变性基础定理）：**

如果集合边界上的"速度"总是指向内部或沿边界 → 系统永远不会离开集合

```
    安全集 C
    ┌─────────┐
    │  ↑ ↑ ↑  │  边界上速度都指向内部
    │ ←     → │
    │  ↓ ↓ ↓  │
    └─────────┘
```

**递归证明：**

```
已知：ψ_{j,m-1} ≥ 0         （Nagumo 定理）
  ↓ 由定义：ψ̇_{j,m-2} + α_{m-1}(ψ_{j,m-2}) = ψ_{j,m-1} ≥ 0
推出：ψ_{j,m-2} ≥ 0         （再次 Nagumo）
  ↓ ...递归...
最终：ψ_{j,0} = b_j(x) ≥ 0  （原始安全约束满足！） ■
```

## 第27页：安全性定理 3.2——ABNet 合并

**定理 3.2：** 两个 ABNet 再次用凸组合合并后仍保证安全

**递归合并：**

```
ABNet₁ = 头₁ + 头₂ + ... + 头ₐ     安全 ✓
ABNet₂ = 头ₐ₊₁ + ... + 头_b        安全 ✓

合并 = w·ABNet₁ + (1-w)·ABNet₂     仍然安全 ✓！
```

**实际意义：**

## 第44页：实验 3——为什么 BNet 有理论保证仍碰撞？

**理论 vs 实践的差距：**

```
理论保证: 连续时间下完美执行 → 安全

实际情况:
  1. 离散时间控制 → 两个时间点之间可能违反约束（采样间效应）
  2. CBF 参数学习不好 → 离散化误差更大
  3. 噪声放大 → 单头模型对噪声敏感 → QP 解剧烈变化
```

**ABNet 如何解决：**
- 多头融合过滤噪声 → 输入更稳定
- 多个头的 QP 解取加权平均 → 减少解的剧烈变化
- 即使某个头因噪声偏差，其他头可以补偿

**关键结论：** 理论保证在理想条件下成立，ABNet 将这种保证更好地延伸到实际条件

## 第47页：局限性与未来方向

**三大局限性：**

| 局限性 | 当前假设 | 未来方向 |
|--------|----------|----------|
| 依赖已知动力学 | $f(\boldsymbol{x}), g(\boldsymbol{x})$ 已知 | 用 Neural ODE 学习动力学 |
| 依赖准确状态 | 状态 $\boldsymbol{x}$ 可准确获取 | 用基础模型从高维观测估计状态 |
| 依赖已知安全规范 | $b_j(\boldsymbol{x}) \geq 0$ 已知 | 从数据中学习安全规范 |

**详细分析：**

1. **动力学依赖：** HOCBF 中的 $L_f, L_g$ 需要精确的 $f, g$ → 动力学不准确时安全保证打折扣 → 解决方案：Neural ODE 同时学习动力学 + 安全控制

2. **状态估计：** 高维观测（图像）到状态的映射本身可能不准 → 解决方案：CLIP/BLIP 等视觉基础模型提供更鲁棒的状态表示

3. **安全规范：** 某些任务难以用数学公式定义"安全" → 解决方案：从专家演示中自动学习安全约束

## 附录页（备用）：指标定义

**SAFETY（公式 16）：**

$$\text{SAFETY} = \min_k \{\min_{t \in [t_0, T]} b(\boldsymbol{x}(t))\}_k$$

所有运行中最坏情况的安全裕度。≥0 → 所有运行都安全

**CONSER.（公式 17）：**

$$\text{CONSER.}_{\text{mean}} = \text{mean}_k \{\min_t b(\boldsymbol{x}(t))\}_k$$

平均最坏安全裕度。太高 = 过度保守

**UNCERTAINTY（公式 18）：**

$$u_i \text{ UNCERTAINTY} = \text{mean}_t \{\text{std}_k \{u_i(t)\}_k\}$$

所有运行间控制的标准差。低值 = 噪声影响小 = 鲁棒
