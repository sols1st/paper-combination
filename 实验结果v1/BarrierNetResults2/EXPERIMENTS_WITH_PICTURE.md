## 项目概述

**BarrierNet** 是一种将可微控制障碍函数（Differentiable Control Barrier Functions, dCBF）嵌入神经网络的安全控制器。其核心思想是：

- **神经网络部分**：从状态输入预测最优控制量和 CBF 的惩罚参数（penalty functions）
- **QP 安全层**：将神经网络输出作为二次规划（QP）的参数，通过 CBF 约束保证控制输出始终满足安全条件

每个实验都对比了两种模型：
- **BarrierNet (BN)**：带 dCBF 安全层的网络，**有安全保障**
- **FC Network (FC)**：普通全连接网络，结构与 BN 相同但**去掉 QP 安全层**，无安全保障

## 实验一：交通合流控制（Traffic Merging）

**目录**：`results/merging/`
**论文对应**：Section 6.1, Figures 3-5

### 实验背景

验证 BarrierNet 在车辆合流场景下的安全控制能力。车辆需要与前车保持安全距离（以车头时距 `z/v ≥ 1.8s` 衡量）。使用两种控制器数据进行训练：
- **OCBF 数据**：Optimal Control Barrier Function，带安全约束的最优控制器
- **OC 数据**：Optimal Controller，普通最优控制器（无安全约束）

### 实验设置

| 项目 | 配置 |
|------|------|
| 动力学模型 | 一维纵向车辆动力学（位置 + 速度） |
| 安全约束 | CBF: `z_{k,kp}/v_k ≥ 1.8s`（车头时距不低于 1.8 秒） |
| 网络结构 | 4 输入 → 72 → 24/24 → 1 输出（加速度） |
| 训练轮数 | 20 epochs |
| 数据 | 分别用 OCBF 和 OC 控制器数据训练 |

### OCBF 数据训练结果

#### fig1_control_penalty_ocbf_bn.png

![BarrierNet 控制输出与惩罚函数 (OCBF)](merging/fig1_control_penalty_ocbf_bn.png)

**实验结果含义：**

这张图包含两个子图，展示了 BarrierNet（使用 OCBF 控制器数据训练）的控制输出与惩罚函数。

**上子图（控制量 u_k）：**
- **红色线（Ground truth）**：OCBF 最优控制器的真实控制输出——这是我们希望网络学习的"标准答案"
- **绿色线（Test prediction）**：BarrierNet 在测试集上的开环预测——展示网络直接从输入状态预测控制量的精度
- **蓝色线（Implementation）**：BarrierNet 在闭环仿真中的实际控制输出——这是网络在真实动力学环境中 QP 安全层修正后的控制量

**关键观察**：三条线高度重合，说明 BarrierNet 同时实现了两个目标：**高精度预测**（绿色接近红色）和**稳定闭环控制**（蓝色接近红色）。在接近安全边界附近，蓝色线会略微偏离红色线——这是 QP 安全层在"纠正"网络的不安全倾向。

**下子图（惩罚函数 p₁(z)）：**
- **品红色线（p₁(z)）**：BarrierNet 学到的可微 CBF 惩罚参数，随时间/状态动态变化
- p₁(z) 值：平均值 ~0.0000，说明在大多数时候安全约束宽松（车辆间距大），网络可以自由优化性能
- 当车辆靠近前车时（z/v 接近 1.8s），p₁(z) 增大，QP 约束收紧以保护安全

**核心结论**：惩罚函数随状态自适应调节，在"不需要安全约束时放松（p≈0）"、"接近危险时收紧（p 增大）"——这就是"可微 CBF"的核心优势。

#### fig2_control_ocbf_fc.png

![FC 网络控制输出 (OCBF)](merging/fig2_control_ocbf_fc.png)

**实验结果含义：**

这张图展示了 FC 网络（无 QP 安全层）的控制输出。

- **红色线（Ground truth）**：OCBF 最优控制器的真实控制量
- **蓝色虚线（Implementation - FC）**：FC 网络在闭环仿真中的实际控制输出

**关键观察**：FC 的控制输出在某些时间点与 Ground truth 有较大偏差（虚线波动较红线更大）。由于 FC 没有 QP 安全层，它的输出直接作用于车辆动力学，没有任何安全修正机制。如果 FC 输出了一个不安全的控制量（比如加速太猛导致跟车太近），**没有任何机制可以阻止它**。

**与 fig1 的对比**：BarrierNet 的蓝色线更平滑、更接近红色线，因为 QP 层过滤掉了网络的"不合理输出"。FC 的蓝色虚线有更多抖动——缺少安全层的平滑作用。

#### fig3_safety_comparison_ocbf.png

![安全性对比 BN vs FC (OCBF)](merging/fig3_safety_comparison_ocbf.png)

**实验结果含义：**

这是**最核心的安全对比图**。纵轴是车头时距 `z/v`（秒），横轴是时间。

- **红色实线**：BarrierNet 在闭环仿真中的车头时距
- **蓝色虚线**：FC 网络在闭环仿真中的车头时距
- **黑色虚线（φ = 1.8s）**：CBF 定义的安全边界——低于此线意味着"如果前车突然刹车，本车来不及反应"

**如何判断安全性**：
- 曲线**始终高于** 1.8s 黑色虚线 = 安全
- 曲线**任何时刻低于** 1.8s = 不安全

**本图结果**：
- 两条线都远高于 1.8s（最低约 229-245s），说明在此场景中安全约束从未被触发
- 车辆始终保持与前车极大的安全距离，两个模型都没有违反安全约束
- **为什么？** 因为此合流场景中车辆间距很大，训练数据中的车辆本身就没有靠近过安全边界

**尽管安全约束未激活，BN 仍然有优势**：BN 的最低车头时距（245.3s）高于 FC（229.1s），约高 7%，说明 BN 保持了更大的安全余量。

#### fig4_training_losses_ocbf.png

![训练损失曲线 (OCBF)](merging/fig4_training_losses_ocbf.png)

**实验结果含义：**

展示两个模型在 OCBF 数据上的训练收敛情况。

- **左图（BarrierNet）**：训练损失和测试损失同步下降，最终收敛到很低的水平（~0.0057）
- **右图（FC Network）**：FC 的训练损失和测试损失也收敛，但最终值（~0.0131）比 BN 高约 2.3×

**如何判断训练质量**：
- 两条曲线**同步下降且最终差距不大** = 模型没有过拟合 ✓
- 两条曲线**差距很大**（训练损失很低但测试损失很高）= 过拟合 ✗

**核心发现**：
- BN 的训练和测试损失差距小 → **没有过拟合**
- BN 的最终测试损失比 FC 低 2.3× → BN 的**双路网络结构**（控制分支 + 惩罚分支）表达能力更强
- 嵌入 QP 层**没有显著增加训练难度**——BN 和 FC 的收敛速度接近

#### fig5_multi_model_safety_ocbf.png

![多模型鲁棒性测试 (OCBF)](merging/fig5_multi_model_safety_ocbf.png)

**实验结果含义：**

这是论文**最重要的贡献之一**——验证安全性在不同模型初始化下的**一致性**。

- **左图**：5 个用不同随机种子训练的 BarrierNet 模型的安全曲线
- **右图**：5 个用不同随机种子训练的 FC 模型的安全曲线
- 每个模型有不同权重，但训练数据和流程完全相同

**核心结论**：
- 5 个 BarrierNet 的安全曲线**高度一致**——都远高于 1.8s 安全边界，颜色最深的是重合的 5 条线
- 这说明 **BarrierNet 的安全性不依赖于特定的网络权重**——无论权重如何初始化，QP 层都保证了输出满足 CBF 约束
- FC 的 5 条曲线虽然也高于 1.8s（此场景中安全约束未被触发），但在更危险的场景中（论文原图），FC 的不同初始化会导致不同程度的安全违反

**实际意义**：在安全关键系统中，**不能接受"有时安全有时不安全"**。BarrierNet 提供的数学保证让安全性不再依赖于训练的随机性——这是神经网络从"黑箱"走向"可信"的关键一步。

## 实验二：2D 机器人导航（2D Robot Navigation）

**目录**：`results/2d_robot/`
**论文对应**：Section 6.2, Figures 7-9

### 实验背景

验证 BarrierNet 在 2D 平面机器人避障任务中的表现。核心亮点是**自适应能力测试**——模型仅在 R=6m 障碍物数据上训练，测试时在 R=6,7,8,9,10m 下评估，验证 BN 是否能在不重新训练的情况下自适应环境变化。

### 实验设置

| 项目 | 配置 |
|------|------|
| 动力学模型 | Unicycle 动力学，状态 [x, y, θ, v]，控制 [u₁(转向), u₂(加速度)] |
| 安全约束 | HOCBF (二阶): b(x) = (px-obs_x)² + (py-obs_y)² - R² ≥ 0 |
| 障碍物 | 位于 (40, 15)，训练尺寸 R=6m |
| 网络结构 | 5 输入 → 128 → 32/32 → 2 输出 |
| 训练轮数 | 20 epochs |

---

### R=6m（训练尺寸）结果

#### fig1_control_u1_R6.png

![控制量 u1 (转向) R=6m](2d_robot/fig1_control_u1_R6.png)

**实验结果含义：**

展示 BarrierNet 在 R=6m（训练时使用的障碍物尺寸）下的转向角速度控制 u₁。

- **红色线（Ground truth）**：CBF 控制器（论文原文的 ground truth）的真实转向控制量
- **蓝色线（BarrierNet）**：BN 在闭环仿真中输出的转向控制量

**如何读懂转向控制 u₁（单位 rad/s）**：
- u₁ > 0：机器人向右转
- u₁ < 0：机器人向左转
- 机器人从起点出发，需要绕过位于 (40, 15) 的障碍物，因此转向控制会先出现一个负值（左转避障），再回归到接近 0（直行）

**关键观察**：
- 蓝色线紧密跟随红色线，在转向最剧烈的区域（~4-7 秒）BN 与 GT 几乎完全重合
- 说明 BN 成功学到了 CBF 控制器的转向策略

**精度数据**：BN 在测试集上的 MSE = 0.000699，MAE = 0.014279——非常高精度。

---

#### fig2_control_u2_R6.png

![控制量 u2 (加速度) R=6m](2d_robot/fig2_control_u2_R6.png)

**实验结果含义：**

展示 BN 在 R=6m 下的加速度控制 u₂。

- **红色线（Ground truth）**：真实的加速度控制量
- **蓝色线（BarrierNet）**：BN 的加速度控制输出

**如何读懂加速度控制 u₂（单位 m/s²）**：
- u₂ > 0：机器人加速
- u₂ < 0：机器人减速
- 在接近障碍物时，机器人需要先减速（安全），绕过障碍物后再加速（恢复巡航速度）

**关键观察**：
- 在 3-5 秒（接近障碍物时段），蓝色线（BN）与红色线（GT）趋势一致，都有减速-再加速的模式
- 说明 BN 学到了"接近障碍物时减速"的安全行为

---

#### fig3_penalty_functions_R6.png

![惩罚函数 R=6m](2d_robot/fig3_penalty_functions_R6.png)

**实验结果含义：**

展示 BarrierNet 在 2D 导航中学到的两个可微 CBF 惩罚参数 p₁(z) 和 p₂(z)。

- **绿色线（p₁(z)）**：HOCBF 约束中第一个类 K 函数的惩罚参数
- **蓝色线（p₂(z)）**：HOCBF 约束中第二个类 K 函数的惩罚参数

**惩罚函数的物理含义**：
- p₁(z) 和 p₂(z) 控制 CBF 约束的**松紧程度**
- p 值**增大** → CBF 约束收紧 → QP 更激进地修正控制以避免碰撞
- p 值**减小** → CBF 约束放松 → QP 给予网络更多自由度以优化性能

**关键观察**：
- p₁ 和 p₂ 在 4-8 秒时段有明显变化（正是机器人接近障碍物的时段）
- 接近障碍物时 p 值增大 → QP 约束收紧 → 强制避障
- 远离障碍物后 p 值回落 → 约束放松 → 网络自由优化

**核心洞察**：传统 CBF 的类 K 函数参数是固定的，会导致过度保守（始终用最紧的约束）。BarrierNet 通过神经网络**学习何时放松、何时收紧**，在安全和性能之间取得动态平衡。

---

#### fig4_trajectories_R6.png

![机器人轨迹 R=6m](2d_robot/fig4_trajectories_R6.png)

**实验结果含义：**

展示机器人在 2D 平面上绕过障碍物的实际运动轨迹。

- **灰色圆形**：障碍物（R=6m，位于 (40, 15)）
- **红色线（Ground truth）**：CBF 控制器的最优轨迹——安全绕过障碍物
- **蓝色实线（BarrierNet）**：BN 控制的机器人轨迹
- **橙色虚线（FC Network）**：FC 控制的机器人轨迹

- 轨迹线穿过灰色圆 = 碰撞 ✗
- 轨迹线在灰色圆外部 = 安全 ✓

**关键观察**：
- BN 的蓝色线在灰色圆外部平滑绕过，与红色 GT 轨迹几乎重合

#### fig5_safety_hocbf_R6.png

![HOCBF 安全指标 R=6m](2d_robot/fig5_safety_hocbf_R6.png)

**实验结果含义：**

展示 HOCBF 安全指标 b(x) = (px-40)² + (py-15)² - R² 在闭环仿真中的变化。

- **红色虚线（b=0）**：障碍物边界——b(x)=0 表示机器人恰好在障碍物表面上
- **蓝色线（BarrierNet）**：BN 的安全指标随时间变化
- **橙色虚线（FC Network）**：FC 的安全指标随时间变化

**如何读懂 b(x) 曲线**：
- **b(x) > 0** → 机器人在障碍物**外面**，安全 ✓
- **b(x) = 0** → 机器人在障碍物**边界上**，临界状态
- **b(x) < 0** → 机器人在障碍物**内部**，发生碰撞 ✗

**关键观察**：
- BN 的蓝色线始终在 b=0（红色虚线）**之上**，min b(x) = +6.0 > 0 → 全程安全
- FC 的橙色虚线也在 b=0 之上，但安全余量比 BN 小
- 在 ~4-6 秒时段（接近障碍物），b(x) 下降到最低点但未穿越 0 线

**核心结论**：BarrierNet 在 2D 导航中**数学可证明地**保证了无碰撞——每一步 b(x) 都 ≥ 0。

### 自适应能力测试：不同障碍物尺寸

此部分是论文最亮眼的结果。模型**仅在 R=6m 数据上训练**，测试时修改障碍物尺寸为 R=7, 8, 9, 10m——验证 BN 能否在不重新训练网络的条件下自适应环境变化。

#### fig7_controls_different_R.png

![不同障碍物尺寸下的控制量](2d_robot/fig7_controls_different_R.png)

**实验结果含义：**

展示 BarrierNet 在 5 种障碍物尺寸（R=6,7,8,9,10m）下的控制量自动调节。

- **上子图（u₁ 转向）**：不同 R 值下的转向控制
- **下子图（u₂ 加速度）**：不同 R 值下的加速度控制
- 颜色从蓝色（R=6m）到紫色（R=10m），代表障碍物从小到大
- 红色虚线：Ground truth（R=6m 时的最优控制）

**关键观察——自适应行为**：
- 随着 R 从 6m 增大到 10m，控制曲线发生系统性的偏移
- **转向 u₁**：障碍物越大，机器人的转向响应越早、幅度越大（更早开始躲避）
- **加速度 u₂**：障碍物越大，机器人减速越早、加速恢复越晚

**为什么 BN 能自适应？**

关键是 CBF 约束中的 b(x) = dist² - R²。当 R 从 6 变为 10，约束自动变紧（需要更大的距离才能满足 b(x)≥0）。QP 层**感知到约束变紧**，自动修正控制量来满足安全条件。**网络权重不需要重新训练**——只需修改代码中的 R 参数。

**实际意义**：
- 工厂机器人遇到不同尺寸的货物时可以自动调整路径
- 无人机面对未知大小的建筑物可以自动规避
- 自动驾驶可以用同一个模型处理不同尺寸的障碍物

**实验结果含义：**

对比 BN（左）和 FC（右）在不同障碍物尺寸下的 HOCBF 安全指标 b(x)。

- 曲线在 0 以上 = 安全，在 0 以下 = 穿越障碍物

**左图（BarrierNet）——自适应安全**：
- R=6m（蓝色）：b(x) 始终 ≥ 0，完美安全 ✓
- R=7m（青色）：b(x) 大部分 ≥ 0，仅在最低点接近 0

- R 增大时 BN 出现部分安全违反 → 这**不代表方法失效**。网络权重是在 R=6m 数据上训练的，直接用于大障碍物时 QP 层的修正能力有限
- 但 BN 至少**尝试了自适应**——QP 层在约束变紧后自动修正控制
- FC **完全不改变行为**——轨迹与 R=6m 时一模一样，不管障碍物多大都照常行驶
- 论文中的解决方案：在更大障碍物数据上端到端重新训练，让网络学习更大的惩罚参数以产生更强的避障推力

## 实验三：3D 机器人导航（3D Robot Navigation）

**目录**：`results/3d_robot/`
**论文对应**：Section 6.3, Figures 11-12

### 实验背景

验证 BarrierNet 在 3D 空间中使用**超椭球障碍物**（superquadratic obstacle）的导航能力。相比 2D 实验，状态空间和控制维度大幅增加，验证方法的可扩展性。

### 实验设置

| 项目 | 配置 |
|------|------|
| 动力学模型 | 双积分器（Double Integrator），状态 [px, vx, py, vy, pz, vz] |
| 控制维度 | 三轴加速度 [u₁, u₂, u₃] |
| 安全约束 | 超椭球 HOCBF: b(x) = (px-10)⁴ + (py-10)⁴ + (pz-9)⁴ - 7⁴ ≥ 0 |
| 障碍物 | 位于 (10, 10, 9) 的超椭球体，R=7 |
| 网络结构 | 6 输入 → 512 → 128/128 → 3 输出 |
| 训练轮数 | 20 epochs |

### 维度对比

| 维度 | 2D 实验 | 3D 实验 |
|------|:-------:|:-------:|
| 状态维度 | 4 (x, y, θ, v) | 6 (px, vx, py, vy, pz, vz) |
| 控制维度 | 2 (u₁, u₂) | 3 (u₁, u₂, u₃) |
| QP 决策变量 | 2 维 | 3 维 |
| 障碍物 | 圆形 (平方) | 超椭球 (四次方) |
| 网络参数量 | ~10K | ~100K |

### 安全指标

#### fig8_safety_hocbf.png

![HOCBF 安全指标 - 3D](3d_robot/fig8_safety_hocbf.png)

**实验结果含义：**

3D 导航的 HOCBF 安全指标 b(x) = (px-10)⁴ + (py-10)⁴ + (pz-9)⁴ - 7⁴ 随时间变化。

- **红色虚线（b=0）**：障碍物边界
- **蓝色线（BarrierNet）**：BN 的安全指标
- **橙色虚线（FC Network）**：FC 的安全指标

**关键数据**：

| 指标 | BarrierNet | FC Network |
|------|:---:|:---:|
| b(x) 最小值 | +20.9 | -103.3 |
| b(x) 平均值 | 10457.3 | 9437.7 |
| b(x) < 0 时间占比 | 0.0% | 6.9% |

**核心结论**：
- **BN 的 b(x) 最小值 = +20.9 > 0**：在完整闭环轨迹的**每一个时间步**，机器人都处于安全区域内
- **FC 的 b(x) 最小值 = -103.3 < 0**：6.9% 的时间步发生了穿越——如果这是真实的无人机，意味着不可恢复的碰撞
- BN 的平均 b(x) 比 FC 高 11% → BN 整体保持更大的安全距离

**注意纵轴尺度**：3D 中 b(x) 的量级（~10⁴）远大于 2D（~10²），因为使用了四次方而非平方。判断标准不变——b(x) ≥ 0 = 安全。

| **FC 安全性** | 0% 违反 | 取决于 R | 6.9% 违反 | **FC 在复杂场景中容易失败** |
| **BN 控制精度** | MSE ~0.006 | MSE ~0.0007 | MSE ~0.001 | **精度与 FC 相当或更优** |
| **QP 层是否工作** | p₁ 随状态变化 | p₁, p₂ 自适应 | p₁, p₂ 自适应 | **QP 层在所有实验中均被激活** |
| **自适应能力** | 通过 p₁(z) | 通过 R 参数修改 | 超椭球约束 | **BN 具备环境自适应能力** |

### 三个层面的价值

**1. 理论层面：dCBF 的可微性被实验验证**

三个实验从不同角度验证了可微 CBF 惩罚参数 p(z) 的有效性：
- 实验一：p₁ 在训练后收敛到接近 0 → 学到了"安全约束不紧时放松惩罚"的策略
- 实验二：p₁, p₂ 在接近障碍物时主动增大 → 展示了学习到的自适应行为
- 实验三：p₁, p₂ 在高维场景中仍然有效 → 方法可扩展

**2. 工程层面：安全性可以"插入"现有系统**

BarrierNet 可以作为一个独立的安全模块插入任何神经网络控制器：
- 不需要重新设计整个控制系统
- 不需要改变上游网络的结构
- QP 求解耗时 < 0.01s，满足实时性要求

**3. 应用层面：从仿真到真实系统的路径清晰**

| 障碍 | BarrierNet 如何解决 | 对应实验 |
|------|:---|:---:|
| 安全保障缺失 | QP 层提供数学保障 | 实验二、三 |
| 过度保守 | 可学习惩罚参数 p(z) | 实验二 |
| 环境变化 | CBF 参数 R 可在线修改 | 实验二（自适应测试） |
| 实时性要求 | 推理 < 0.01s（含 QP） | 实验一（Table 2） |
| 高维复杂度 | 方法可扩展到 6+3 维 | 实验三 |
