# SafePVC 实验结果分析报告

## 1. 实验概述

### 1.1 实验背景

本实验基于 **SafePVC (Provably Probabilistic Safe Controller Synthesis)** 框架，针对 **CARLA 自动紧急制动 (AEBS)** 基准进行测试。该框架的核心目标是：为基于视觉的神经网络控制系统 (NNCS) 提供 **形式化的概率安全保证**。

### 1.2 实验流程

SafePVC 主循环 (`Aebs.VT.loop`) 实现了论文 Algorithm 1 的 **CEGIS (Counterexample-Guided Inductive Synthesis)** 范式：

```
┌─────────────────────────────────────────────────────┐
│  1. 训练 Lyapunov 函数 (L-Net)  → 10 epochs         │
│  2. 验证下降条件 (check_dec_cond)                    │
│  3. 如果满足 → 计算概率安全界                         │
│  4. 训练控制器 (P-Net) → 1 epoch                     │
│  5. 迭代直到收敛或超时                                │
└─────────────────────────────────────────────────────┘
```

### 1.3 系统状态空间

| 维度 | 物理含义 | 范围 |
|------|---------|------|
| d (归一化距离) | 与前车的距离 | [min(y_data), max(y_data)] |
| v (速度) | 自车速度 | [0.0, 3.0] m/s |

- **初始集合 (Init)**: d ∈ [15.0/σ, 16.0/σ], v ∈ [2.5, 3.0] （远距离、高速状态）
- **不安全集合 (Unsafe)**: d ∈ [5.0/σ, 6.0/σ], v ∈ [0.5, 3.0] （近距离、未充分减速）
- **动作空间**: 加速度 acc ∈ [-3.0, 3.0] m/s²

## 2. 实验参数配置

### 2.1 核心参数（当前设置）

| 参数 | 值 | 说明 |
|------|-----|------|
| **L-Net 结构** | [2, 16, 8, 1] | 2层隐藏层，tanh 激活，softplus 输出 |
| **L-Net 学习率** | 3e-3 | Adam 优化器 |
| **P-Net 学习率** | 5e-2 | Adam 优化器 (仅训练 controller_net) |
| **P-Net Lipschitz 目标 (p_lip)** | 2.0 | 控制器梯度范数上限 |
| **L-Net Lipschitz 目标 (l_lip)** | 4.0 | Lyapunov 函数梯度范数上限 |
| **ε (martingale eps)** | 0.1 | 鞅下降条件的松弛量 |
| **γ_decrease** | 1.0 | 衰减因子 (=1.0 使用 eps 松弛) |
| **reach_prob (目标)** | 0.95 | 期望达到的安全概率下界 (95%) |
| **grid_size (space_split)** | [100, 100] | 验证网格分辨率 = 10,000 个格点 |
| **batch_size** | 2048 | 训练和验证的批大小 |
| **K (Lipschitz 常数上界)** | 1.2 | 用于 Lipschitz 修正 |
| **noise_factor** | 0.05 | 噪声幅度 = 状态空间范围 × 5% |
| **violation_threshold** | 0.001 | 硬违反比例阈值 (≤0.1% 视为满足) |
| **max_iterations** | 100 (loop.py) | 最大迭代次数 |
| **timeout** | 3600s (1小时) | 超时时间 |

### 2.2 训练策略

每次迭代中：
- **L-Net 训练**: 10 epochs，使用鞅损失 (martingale loss) + Lipschitz 正则 + 区域损失
- **P-Net 训练**: 1 epoch，使用鞅损失 + Lipschitz 正则 + MSE 监督损失（以原始 PPO 控制器为教师网络）
- **噪声采样**: 三角分布 (triangular distribution)，每样本 16 个噪声采样 (L-Net) / 128 个 (P-Net)

## 4. 实验效果评价

### 4.1 积极方面 ✅

1. **框架有效性得到验证**: 运行 8 和运行 2 分别达到了 95.691% 和 95.711% 的安全概率下界，超过了论文设定的 95% 目标阈值
2. **鞅损失收敛良好**: `dec_loss_l` 从初始的 ~0.10 稳定下降到 ~0.008-0.015，表明 L-Net 逐渐满足超鞅条件
3. **硬违反控制出色**: 大多数迭代的硬违反数为 0-12/10000 (≤0.12%)，远低于 0.1% 阈值的边界
4. **训练效率**: 单次迭代约 1 秒，100 次迭代约 100 秒，计算代价可接受

### 4.2 存在的问题 ⚠️

1. **成功率不稳定**: 8 次运行中仅 1 次获得有效概率界 (12.5% 成功率)，说明框架对随机种子敏感
2. **"RSM is lower at unsafe than in init"**: 这是最常见的问题 — Lyapunov 函数在不安全区域的值无法超过初始区域，导致无法构造有意义的概率界
3. **概率界波动大**: 即使收敛后，概率界在 65%-96% 之间大幅波动，说明 L-Net 的稳定性不足
4. **长训练不一定有效**: 运行 7 (1000 次迭代) 仍然 UNSAFE，说明简单的迭代增加不能解决问题
5. **L-Net Lipschitz 损失**: 在后期训练中 `lip_loss_l` 增大到 5.0-7.5，说明 L-Net 的梯度范数频繁超过目标值 4.0

### 4.3 与论文声称的对比

| 指标 | 论文目标 | 实验实际 | 差距 |
|------|---------|---------|------|
| 安全概率下界 | ≥95% | 最高 95.711% | ✅ 达标 (但不稳定) |
| 收敛速度 | — | ~35-130 次迭代 | 需要较长 warmup |
| 成功率 | — | 12.5% (1/8) | 需改善 |

## 5. 可调参数详解与调优建议

### 5.1 网络结构参数

#### `l_model_config` — Lyapunov 网络结构
```python
# 当前: [2, 16, 8, 1]
# 建议尝试:
l_model_config = [2, 32, 16, 1]   # 增大网络容量
l_model_config = [2, 64, 32, 16, 1]  # 更深网络
```
- **影响**: 更大的网络可以更好地拟合复杂的 Lyapunov 函数，但会增加 IBP 验证的保守性
- **权衡**: 网络越大，auto_LiRPA 的区间界越松，验证越难通过

#### `square_l_output` — L-Net 输出非负化
```python
# 当前: True (使用 softplus)
# 替代方案:
# - x**2 (平方输出)
# - F.relu(x) (ReLU 截断)
```
- **影响**: 保证 L-Net 输出非负，是鞅理论的必要条件
- **建议**: softplus 比平方更平滑，梯度更稳定

### 5.2 训练超参数

#### `eps` — 鞅松弛量
```python
# 当前: 0.1
# 建议范围: [0.01, 0.5]
```
- **含义**: 允许 E[L(s_{t+1})] ≤ L(s_t) + ε，松弛超鞅条件
- **调大**: 更容易满足下降条件，但概率界会变松
- **调小**: 概率界更紧，但训练更困难

#### `gamma_decrease` — 指数衰减因子
```python
# 当前: 1.0 (不衰减)
# 建议范围: [0.9, 0.99]
```
- **含义**: 要求 E[L(s_{t+1})] ≤ γ·L(s_t)，当 γ<1 时强制指数衰减
- **调小**: 更强的衰减要求，理论上更紧的概率界，但训练难度大增

#### L-Net / P-Net 学习率
```python
# 当前: L-Net lr=3e-3, P-Net lr=5e-2
# 建议:
l_optimizer = Adam(lr=1e-3)   # 降低 L-Net 学习率以提升稳定性
p_optimizer = Adam(lr=1e-2)   # 降低 P-Net 学习率以减少波动
```
- **P-Net 学习率 (5e-2) 偏高**: 这是概率界波动的主要原因之一
- **建议**: 使用学习率调度器 (如 CosineAnnealingLR)

#### `lip_coeff` — Lipschitz 正则化系数
```python
# 当前: 0.001
# 建议范围: [0.01, 0.1]
```
- **影响**: 控制 Lipschitz 约束的强度
- **当前值偏小**: 后期 `lip_loss_l` 高达 5-7，说明 Lipschitz 约束未被有效执行
- **建议**: 增大到 0.01-0.05，或使用渐进式增大策略

## 6. 优先级调优建议

基于实验结果，以下是最可能改善效果的调优方向（按优先级排序）：

| 优先级 | 参数/策略 | 当前值 | 建议值 | 预期效果 |
|:------:|----------|-------|-------|---------|
| 🔴 P0 | P-Net 学习率 | 5e-2 | 1e-2~2e-2 | 减少概率界波动 |
| 🔴 P0 | lip_coeff | 0.001 | 0.01~0.05 | 增强 Lipschitz 约束 |

## 7. 结论

本实验在 CARLA AEBS 基准上验证了 SafePVC 框架的有效性。最佳运行达到了 **95.7%** 的概率安全下界，满足论文设定的 95% 目标。然而，框架的成功率较低 (1/8)，且概率界在训练后期存在较大波动。主要瓶颈在于：

1. **L-Net 无法可靠地拉开初始区域与不安全区域的函数值差距**
2. **P-Net 学习率过高导致训练不稳定**
3. **Lipschitz 正则化强度不足**

建议按上述优先级调优，特别关注 P-Net 学习率降低和 Lipschitz 系数增大，这两个改动成本最低但预期收益最大。
