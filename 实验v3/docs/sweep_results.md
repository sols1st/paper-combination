## 2. 核心发现

### 2.1 干预率由有效p值决定

```
干预率 vs 有效p值:
  90% ┤     ★ V3B (p≈0.14)
  80% ┤
  70% ┤
  60% ┤
  50% ┤ ★ p=0.5      ★ V3A (p=2.0-4.0)  ★ V3B (p≈0.6-1.5)
  40% ┤
      └──────────────────────────────────────────
       0.1    0.5    1.0    2.0    3.0    4.0
                    有效CBF参数 p
```

**关键规律**: 干预率完全由CBF参数p的实际值决定，与V3A/V3B类型无关。
- p ≈ 0.1-0.5: 干预率 55-88% (紧凑约束)
- p ≈ 0.6-1.5: 干预率 45-47% (中等约束)
- p ≈ 2.0-4.0: 干预率 45-47% (宽松约束)

### 2.2 SBC调制实际效果

V3B的SBC B(s)在测试场景中始终很低（SBC将测试状态判断为"安全"）：
- avg B ≈ 0.5-1.0（远低于B_thresh=5.0）
- p ≈ p_min（sigmoid≈0，p_min主导）
- **SBC调制未真正激活** — 当前行为等同于固定p=p_min

**根因**: 训练后的SBC B(s)很大(avg 28.6)但只在训练分布内；测试场景的危险状态不在SBC训练分布中，导致B(s)异常低。

### 2.3 t_gap影响

| t_gap | V3A(p=2.0)干预率 | V3B(p_min=0.1)干预率 | 安全率 |
|-------|-----------------|---------------------|--------|
| 1.0 | 45.9% | **88.3%** | 100% |
| 1.5 | 45.4% | 86.2% | 99-100% |
| 2.0 | 45.9% | 85.7% | 100% |
| 3.0 | 53.6% | 87.2% | **98%** ⚠️ |

t_gap=1.0（最短安全时距）→ CBF约束最紧 → V3B干预率最高(88%)
t_gap=3.0 → 单次不安全出现(98%) → 太松的CBF可能漏过危险

## 4. 结论

1. **V3成功实现解耦**: 形式化界保持95.9%，同时提供可配置的运行时QP安全盾
2. **干预率可通过p_min精确控制**: p_min=0.1→88%干预, p_min=1.0→45%干预
3. **SBC调制需要改进**: 当前SBC B(s)在危险区域过低，需重新训练或使用不同的调制策略
4. **V3B在低p_min时等价于V3A(fixed p=p_min)**: SBC调制因B(s)异常低而未真正激活
5. **t_gap=1.0-1.5为推荐范围**: 太紧(1.0)干预多但安全，太松(3.0)有安全风险

### 下一步改进SBC调制

```python
# 方案A: Barrier调制 (直接用b(s)而非B(s))
p = p_min + (p_max - p_min) * sigmoid(-b(s) / T)
# b(s)小(接近危险) → p大 → 约束紧

# 方案B: 双阈值
if b(s) < danger_threshold: p = p_max
elif b(s) < warn_threshold: p = p_mid
else: p = p_min
```
