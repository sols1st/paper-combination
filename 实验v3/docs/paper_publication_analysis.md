## 2. 差距分析: B 档需要什么

### 2.1 最需要的: 复杂场景

当前 AEBS 最致命的问题是 **PPO 太安全了** — 不加 QP 也是 0% 违例。评审必问: "如果 Baseline 已经完美, QP 的价值在哪里?"

**解决方案**: 换到 2D 机器人避障场景, 让 PPO 不再完美。

```
AEBS (当前):
  控制: 1D (加速度)
  动力学: 近线性
  PPO 安全表现: 0% 违例
  QP 价值: 只能靠人造故障证明

2D 机器人避障 (扩展):
  控制: 2D (角速度 + 加速度)
  动力学: 非线性 (sin/cos)
  PPO 安全表现: 可能有 5-15% 违例
  QP 价值: 自然地降低违例率 → 有说服力!
```

### 2.2 次需要的: 理论深化

- Lipschitz 理论上界: 证明 `Lip(π_QP) ≥ Lip(π_direct)`
- SBC→CBF 调制的收敛性分析
- 解耦方案的形式化正确性证明

### 2.3 可选的: 真实场景

- CARLA 仿真器: 视觉输入 → 完整感知链 → SBC + QP
- 真实机器人: 2D 机器人硬件实验

## 3. 复杂场景扩展方案

### 3.1 SBC 的瓶颈: IBP 维数灾难

IBP 需要在**整个状态空间网格**上做区间传播:

| 场景 | 状态维度 | 网格 | 单元数 | 可行性 |
|------|:---:|------|:-----:|:---:|
| AEBS | 2D | 100² | 10,000 | ✅ < 1s |
| 2D 机器人 | 4D | 50⁴ | 6,250,000 | ⚠️ 可能, ~10min |
| 2D 机器人 | 4D | 30⁴ | 810,000 | ✅ ~1min |
| CARLA | 6D | 50⁶ | 15,625,000,000 | ❌ 不可行 |

**核心约束**: SBC 只能在 ≤ 4 维状态空间上做 IBP 验证。对于更高维场景, 必须降维。

### 3.2 方案A: 2D 机器人 + SBC (★ 推荐)

**思路**: 将 BarrierNet 的 2D 机器人场景加 SBC 验证。

```
场景: 2D 机器人避障
  状态: [px, py, θ, v] (4D)
      px, py: 位置 (可用 dx, dy 相对障碍物降维)
      θ: 朝向角
      v: 速度
  控制: [ω, a] (2D: 角速度 + 加速度)
  动力学: 非线性 (含 sin/cos)
  CBF: 避障 HOCBF (相对度2)
  安全集: 到障碍物距离 > R
```

**方案A 的优势**:

1. **PPO 不再完美**: 2D 控制 + 非线性的避障任务, PPO 会有一定违例率
2. **QP 自然体现价值**: 不需要人造故障, QP 直接降低真实违例
3. **展示完整的 v3**: SBC 离线验证 + QP 在线防护, 两层都有效
4. **BarrierNet 代码已有**: 2D_Robot/ 代码可直接复用

**关键挑战**:

```
1. IBP 对非线性动力学的处理
   动力学含 sin(θ), cos(θ) → IBP 包围会变松
   对策: 用 CROWN 替代 IBP (auto_LiRPA 已支持, 更紧包围)

2. 2D 控制的 SBC
   controller 有 2D 输出 [ω, a]
   SBC 需要联合验证两者
   对策: 复用现有多维 VTVerifier, 无需架构改动

3. PPO 训练 2D 控制器
   需要定义奖励函数 (到达目标 + 避开障碍)
   对策: 参考 BarrierNet 的专家数据

4. SBC 网格大小
   4D 网格: 30⁴ = 810K 单元 → 每轮验证 ~1-2min, 可接受
```

**工作量**: ~2天

| 步骤 | 估计时间 | 说明 |
|------|:------:|------|
| PPO 训练 2D 控制器 | 4h | 定义环境 + 训练 |
| 构建 SBC/IBP for 2D | 8h | 核心难点: IBP 在非线性动力学上 |
| 实验 + 调参 | 4h | 跑 5 种配置 |
| 论文图表 | 2h | |
| **总计** | **~18h** | |

**预期结果**:

```
配置          SBC概率界   CBF安全率   真实违例率
Baseline      N/A         N/A         5-15% (PPO不完美)
V2 (QP嵌入)   退化        99%+         < 2%
V3A (解耦)    90%+        100%         < 2%
V3D (自适应)  90%+        100%         < 2%

关键: QP 将违例率从 5-15% 降到 <2%
      SBC 同时提供 90%+ 概率界
      → 两类安全机制都体现了可量化的价值!
```

### 3.3 方案B: AEBS 增强版 (低风险)

**思路**: 保持 2D 状态, 在 AEBS 基础上增强。

```
增强:
  + 前车变速 (leader speed profile)
  + 多安全约束 (距离 + 速度限制 + 能量)
  + 更真实噪声模型
  + 可选的视觉管线 (gen_net + state_net)

优势:
  - SBC 验证零风险 (2D)
  - 增加场景复杂度但不影响 IBP

劣势:
  - 仍然 1D 控制
  - PPO 可能仍然很安全
```

### 3.4 方案C: CARLA 高维 + SBC 低维投影

**思路**: 在高维场景运行 QP, 在低维子空间做 SBC。

```
CARLA 完整场景 (6D 状态 + 2D 控制):
  - QP 处理: 避障 CBF + 车道保持 CBF (2个约束)
  - SBC 在低维投影验证: 只验证 [d, v] 子空间
     → "至少碰撞安全有概率保证"

优势:
  - 完整视觉管线 (CNN + LSTM)
  - 真实自动驾驶场景

劣势:
  - SBC 只覆盖部分安全属性
  - 高维 + 低维的 gap 需要论证
  - 实现复杂度最高
```
