# 实验v3: 解耦双重安全 — QP推理时安全盾

> **核心思路**: QP只在推理时作为安全盾使用，不参与训练，彻底解决v2中QP干扰SBC训练的问题

---

## 1. v2 失败根因分析

实验v2将QP层嵌入训练循环，导致形式化概率界下降~8% (95.8% → 87.9%)：

| 问题 | 原因 | 机制 |
|------|------|------|
| **Lipschitz增大** | QP层增加闭环非线性 | SBC验证的K值增大，验证更保守 |
| **CBF vs SBC冲突** | CBF追求即时安全，SBC追求期望递减 | 两个目标优化方向不完全一致 |
| **梯度噪声** | qpth隐式微分在约束边界处不稳定 | 控制器训练受干扰 |
| **反例减少** | QP阻止不安全行为 → SBC反例减少 | CEGIS训练效率下降 |

**核心洞察**: QP的运行时安全价值和SBC的形式化验证价值应该**解耦**——训练时不用QP（保持SBC质量），推理时用QP（提供运行时安全）。

## 2. v3 架构: 解耦双重安全

### 2.2 关键创新

| 创新点 | v2 (训练时QP) | v3 (推理时QP) |
|--------|-------------|-------------|
| QP参与训练 | ✓ (干扰SBC) | ✗ (不影响SBC) |
| SBC形式化界 | 87.9% (下降8%) | **预期95%+** (同baseline) |
| 运行时安全 | ✓ CBF-QP | ✓ CBF-QP (同v2) |
| **SBC→CBF协同** | ✗ 无 | **✓ B(s)调制p参数** |

| 训练速度 | 慢25% | 同baseline |

### 2.3 SBC调制CBF参数 (★ 核心创新)

v2中CBF参数p由神经网络学习，与SBC无关。v3中p由SBC值B(s)直接计算:

$$p(s) = p_{min} + (p_{max} - p_{min}) \cdot \sigma\left(\frac{B(s) - B_{thresh}}{T}\right)$$

其中:
- $p_{min} = 0.1$: 安全时的最小CBF参数（松弛约束）
- $p_{max} = 4.0$: 危险时的最大CBF参数（收紧约束）
- $B_{thresh} = \frac{1}{1-p_{target}}$: SBC的安全阈值
- $T$: 温度参数，控制过渡平滑度

**物理意义**:
- 当B(s)很小（非常安全）→ p ≈ p_min → CBF约束很松 → 控制器自由度大
- 当B(s)接近阈值（危险）→ p ≈ p_max → CBF约束很紧 → 强制推回安全区
- SBC提供"全局安全地图"，CBF根据地图自动调节防御力度

## 3. 实施计划

### 步骤1: 训练Baseline控制器+SBC (与v2 baseline完全相同)
```
- 加载gen_net, state_net, PPO policy
- 训练SBC + 交替训练controller (无QP)
- 验证SBC条件, 记录概率界
```

### 步骤2: 构建推理QP安全盾
```
- 加载训练好的baseline controller + SBC
- 构建QP盾: 接收u_ref, 输出u_safe
- 支持两种模式:
  a) 固定p (v3A)
  b) SBC调制p (v3B)
```

### 步骤3: 对比评估
```
- 50条轨迹 × 4种配置 (B, V2, V3A, V3B)
- 记录: 安全违反率, 安全裕度, 控制平滑度, QP干预率, p值分布
```
