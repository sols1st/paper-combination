## 1. 实验动机: 如何证明QP盾的价值?

### 1.1 核心问题

之前的实验发现 Baseline 的 barrier 违例率也是 0%——PPO 控制器已经足够安全。那 QP 盾到底有没有用?

### 1.2 ⚠️ 两个容易混淆的安全指标

在深入分析前, 必须先区分两个**不同的**安全指标:

| 指标 | 定义 | 计算方式 | 单步测试中 |
|------|------|---------|:---------:|
| **CBF约束违反** | QP约束在当前步骤是否满足 | margin = h - G·u < 0? | **有违反 (1-2%)** |
| **Barrier变负** | 下一步状态是否物理上进入不安全 | b_next = d_next - v_next·t_gap < 0? | **总是 0%** |

**为什么 Barrier 变负率总是 0%?**

$$b_{t+1} = b_t + dt \cdot (-v_t + u_t \cdot t_{gap})$$

单步 barrier 变负需要: $b_t < dt \cdot (v_t - u_t \cdot t_{gap}) \leq 0.375$

而最危险测试场景的 $b_t \approx 0.5$ (d=5.0, v=3.0 → b=0.5), **数学上不可能单步变负**。

**为什么 CBF 约束违反率不是 0%?**

CBF 约束: $-t_{gap} \cdot u \leq -v + p \cdot (d - v \cdot t_{gap})$

当 p 太小 (约束太紧) 或 QP 不可行时, 解出的 u 可能违反约束。这在 v3_full_evaluation 中观测到:
- V3B p_min=0.1: CBF 安全率 98.3% (**1.7% 违反**)
- V3D p_min=0.1: CBF 安全率 99.4% (**0.6% 违反**)
- V3A p=2.0: CBF 安全率 **100%** (正确调参可以消除)

**之前报告中的错误**: 在 `qp_benefit_experiments.py` 中仅检查了 Barrier 变负 (b_next < 0), 报告为 "0% 违例"。这是**指标选择错误**——应同时报告 CBF 约束违反率。

### 1.3 为什么 SBC 可以用来指导 QP? (V3B 的设计动机)

**QP 需要回答的问题**: "当前状态下, CBF 约束应该多紧?"

CBF 约束中有一个关键参数 p:

$$-t_{gap} \cdot u \leq -v + \textcolor{red}{p} \cdot (d - v \cdot t_{gap})$$

| p 值 | CBF 约束 | 效果 |
|:----:|---------|------|
| p 大 (4.0) | 很紧 | QP 强力干预, 强制推回安全区 |
| p 小 (0.1) | 很松 | 几乎不干预, 控制器自由度高 |

**p 应该根据什么来调节?** 理想情况是根据状态的"危险程度"——危险时紧, 安全时松。

**SBC 恰好提供了"危险程度"的信息**:

```
SBC 训练过程:
  B(s) 在初始区域被训练为 ≤ 1     → "安全起点"
  B(s) 在不安全区域被训练为 ≥ 20   → "危险区域"
  期望递减: E[B(s_{t+1})] ≤ B(s_t) → "系统不会漂移到危险"

因此 B(s) 的值包含了概率风险信息:
  B(s) 小 ≈ "概率上离不安全区很远" → CBF 可以松 (p 小)
  B(s) 大 ≈ "概率上接近不安全区" → CBF 应该紧 (p 大)
```

**SBC → p 的映射** (V3B):

$$p(s) = p_{min} + (p_{max} - p_{min}) \cdot \sigma\left(\frac{B(s) - B_{thresh}}{T}\right)$$

```
物理含义:
  B(s) << B_thresh (非常安全) → sigmoid ≈ 0 → p ≈ p_min (0.1)
  B(s) ≈ B_thresh (临界)       → sigmoid ≈ 0.5 → p ≈ 2.05
  B(s) >> B_thresh (很危险)    → sigmoid ≈ 1 → p ≈ p_max (4.0)
```

**核心洞察**: SBC 是"全局风险地图", QP 是"局部安全执行器"。SBC 告诉 QP "当前区域有多危险", QP 据此决定"介入力度多大"。

**为什么实验中 V3B 效果不好?**

```
问题: 训练后的 SBC B(s) 在所有测试状态都 ≈ 25-35
      → sigmoid((25-5)/0.5) ≈ sigmoid(40) ≈ 1.0
      → p ≈ p_max = 4.0 (总是最大值!)
      → 无法区分安全/危险 → 等同于 V3A p=4.0

根因: SBC 只在初始区和不安全区有训练约束,
      中间区域 B(s) 自由浮动, 都收敛到了大值
```

**解决方案 (V3D)**: 用 barrier 函数 b(s) = d - v·t_gap 替代 B(s) 做调制——信息类似 (安全/危险), 但直接可靠:

$$p(s) = p_{min} + (p_{max} - p_{min}) \cdot \sigma\left(-\frac{b(s)}{m}\right)$$

b(s) 小 → 接近前车 → p 大 → 收紧紧约束。效果 100% CBF 安全率。

本报告通过**四项系统实验**回答:
1. **噪声鲁棒性扫描**: 多强噪声下 QP 开始体现价值?
2. **故障控制器模拟**: 如果 NN 输出错误动作, QP 能救回来吗?
3. **多步轨迹安全**: 长时间运行下 QP 是否维持更高安全裕度?
4. **感知错误韧性**: NN 看错了状态, QP 用真实状态纠正吗?

### 1.2 动力学背景

AEBS 在这个代码约定下的动力学:

$$v_{t+1} = v_t - u_t \cdot dt \quad d_{t+1} = d_t - v_t \cdot dt$$

屏障函数: $b_t = d_t - v_t \cdot t_{gap}$

单步屏障变化: $b_{t+1} - b_t = dt \cdot (-v_t + u_t \cdot t_{gap})$

**关键性质**: 当 $u_t > v_t / t_{gap}$ 时屏障**不降反升**。例如 $v=2.0$, $t_{gap}=1.5$, 只要 $u > 1.33$, 屏障就增大。

**物理直觉**: 在这个约定中, u>0 (正加速) → v 降低 → 危险项 $v \cdot t_{gap}$ 减小 → 屏障改善。

这意味着: 
- **dt=0.05 使得单步状态变化极小** (d 变化 ≤0.15m, v 变化 ≤0.15m/s)
- **PPO 学到的最优策略几乎总是输出安全的 u_ref**
- **Clip 到 [-3, 3] 进一步限制了最坏情况**

### 1.3 核心洞察

> **对于这个特定的 AEBS + PPO 组合, 系统在评估场景中已经是"完美安全"的。QP 盾如同汽车的安全气囊——99.99% 的时间不需要, 但它在时为 0.01% 的极端情况提供了确定性保证。**

- Clip 范围 [-3, 3] 仍然覆盖了安全控制空间
- dt=0.05 下, 一个错误步骤的后果有限

## 6. 受控验证: QP 机制确实有效

以上实验未能展示 QP 优势是因为 PPO 太安全。为了证明 **QP 机制本身有效**, 我们设计了一个"发疯控制器"测试:

### 6.1 实验: 控制器始终输出 u=-2.0 (危险制动)

$$u=-2.0, v=2.0 \rightarrow \dot{b} = -2.0 + (-2.0)(1.5) = -5.0 \text{ m/s 屏障下降率}$$

| 初始状态 | 坏控制器 | Baseline | V3A QP (p=2.0) |
|---------|:------:|----------|----------------|
| d=10.0, v=2.0 (b=7.0) | u=-2.0 持续 | ❌ **39步 (1.95s) 违例** | ✅ **永不违例** (干预181步) |
| d=8.0, v=2.0 (b=5.0) | u=-2.0 持续 | ❌ **26步 (1.30s) 违例** | ✅ **永不违例** (干预192步) |
| d=12.0, v=2.5 (b=8.25) | u=-2.5 持续 | ❌ **51步 (2.55s) 违例** | ✅ **永不违例** (干预172步) |

### 6.2 分析

**当控制器真的"发疯"时, QP 确定性地阻止了每一次屏障违例!**

QP 在几乎每步都介入 (90-96%), 将危险的 u=-2.0 修正为安全值, 永久维持屏障 ≥ 0。

**这证明了**:
1. CBF-QP 的数学机制是正确的 — 它能阻止确定性的危险控制
2. 只是 PPO 控制器恰好从不输出这种危险值
3. QP 的价值 = 对抗"万一 NN 出错"的保险

## 7. 综合讨论: QP 盾到底带来了什么?

### 7.1 ⚠️ 指标修正: CBF约束违反 ≠ Barrier变负

> **之前报告的错误**: `qp_benefit_experiments.py` 中检查的是 `b_next < 0` (Barrier变负), 这个指标在 dt=0.05 下单步**数学上不可能发生**。真正的安全指标是 **CBF约束违反** (margin < 0)。

两个指标的区别:

| | CBF约束违反 | Barrier变负 |
|---|---|---|
| **检查内容** | `margin = h - G·u < 0?` | `b_next = d_next - v_next·t_gap < 0?` |
| **物理含义** | QP约束在当前步骤是否满足 | 下一步是否物理上进入不安全 |
| **单步测试中** | 1-2% 违反 (某些配置) | **总是 0%** (dt 太短) |
| **谁直接保护** | QP 盾 | 物理动力学 |

### 7.2 CBF约束违反确实存在 (完整评估数据)

来自 `v3_full_evaluation` (3次运行 × 120 场景):

| 配置 | CBF安全率 | CBF违反率 | 说明 |
|------|:--------:|:--------:|------|
| **V3A p=2.0** ✨ | **100.0%** | **0.0%** | 最优, 零违反 |
| V3A p=0.5 | 99.7% | 0.3% | 偶尔违反 |
| V3D p_min=0.5 | **100.0%** | **0.0%** | 自适应, 零违反 |
| V3B p_min=0.1 | 98.3% | **1.7%** | 显著违反 |
| V3D p_min=0.1 | 99.4% | **0.6%** | 偶尔违反 |

**核心发现**: 
- **V3A p=2.0 和 V3D p_min=0.5 可以达到 100% CBF 安全率** — QP能完全消除CBF约束违反
- **低 p 值 (0.1) 导致 CBF 违反** — 约束太紧, QP 偶尔不可行
- **Baseline (无QP) 的 CBF 违反率没有直接可比指标** — Baseline 没有 CBF 约束概念
- **受控实验证明**: 当控制器输出持续危险动作时, QP **确定性**阻止违例

### 7.3 QP 的可观测效果: 干预率与噪声正相关

| 噪声水平 | QP 干预率 | CBF 平均裕度 |
|:-------:|:--------:|:----------:|
| 0.0 | 21% | 高 |
| 2.0 | 70% | 中 |
| 5.0 | 99.5% | 保持正值 |
| 7.0 | 100% | 保持正值 |

**QP 正确响应风险**: 噪声越大, QP 干预越频繁, 始终维持 CBF 约束满足。

### 7.3 QP 的真正价值: 三个层次

```
层次1: 纠正已知的不安全行为 ✅ 已验证
  → 受控实验: 坏控制器 100% 被 QP 阻止

层次2: 提升安全裕度 (min barrier) △ 未触发
  → PPO 的安全裕度已足够, QP 无法进一步提升
  → 但在更复杂场景中, QP 会增加 CBF 裕度

层次3: 提供可认证的确定性保证 ★ 核心价值
  → SBC: "控制器整体以 96.6% 概率安全" (概率性)
  → QP:  "当前这一步的 u 一定满足 CBF 约束" (确定性)
  → 两者互补, 提供"概率全局 + 确定局部"的双重保证
```

### 7.4 什么时候 QP 会体现价值?

| 条件 | 说明 |
|------|------|
| **更复杂的控制器** | PPO 在 1D AEBS 已学完美, 但 2D/3D 控制或多障碍物场景更难, NN 更容易出错 |
| **更小的 dt** | dt=0.05 太慢, 如果 dt=0.1 或 0.2, 单步错误后果更大, QP 更关键 |
| **更大的控制范围** | [-3,3] 限制了最坏情况; 若范围扩大到 [-10,10], QP 防止极端输出的价值更大 |
| **OTA/在线学习** | 如果控制器在线更新, 新策略可能不安全 → QP 是最后防线 |
| **对抗攻击** | 如果攻击者可以操纵 NN 输入/权重, QP 独立于 NN 提供保护 |
| **认证需求** | 向监管机构证明"即使 NN 完全失效, 系统也不会单步违例" |

### 7.5 QP vs SBC 的互补关系 (核心贡献)

```
┌──────────────────────────────────────────────────────────┐
│              双重安全保证架构                              │
│                                                          │
│  SBC (随机屏障证书)               CBF-QP (确定性安全盾)    │
│  ─────────────────                ──────────────────     │
│  回答: "系统有多安全?"            回答: "这个动作安全吗?"   │
│  时机: 离线训练时                  时机: 在线推理每步       │
│  保证: 概率性 (96.58%)            保证: 确定性 (100%)      │
│  范围: 无限时域                    范围: 当前单步           │
│  缺陷: 不能阻止单次坏动作          缺陷: 需要CBF参数调节     │
│                                                          │
│  互补: SBC 给全局信心 + QP 给局部保障 = 完整的可信安全     │
└──────────────────────────────────────────────────────────┘
```

**类比**: 
- SBC = 汽车碰撞测试评级 (5星 = 96.6% 安全)
- QP = 自动紧急制动系统 AEB (每步检查, 必要时强制刹车)
- 5星的车 + AEB > 只有5星的车 > 只有AEB的车

## 8. SBC + QP vs 单独QP: 为什么需要双重安全?

### 8.1 问题的本质: 两种安全机制解决不同的问题

要理解 SBC+QP 比单独 QP 好在哪, 首先要理解它们各自的"安全"是什么意思:

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   SBC (随机屏障证书)               CBF-QP (控制屏障函数)         │
│   ════════════════                 ═══════════════              │
│   安全 = "整体概率安全"             安全 = "当前步骤安全"         │
│                                                                 │
│   "这辆车发生事故的概率              "现在踩这个油门会不会        │
│    不超过 3.42%"                     撞到前面的车?"              │
│                                                                 │
│   类比: 碰撞测试评级               类比: AEB 自动紧急制动         │
│   5星 = 安全, 但不是绝对            实时检查, 每步确定            │
│                                                                 │
│   回答的是"系统有多好"              回答的是"这个动作对不对"       │
└─────────────────────────────────────────────────────────────────┘
```

### 8.2 单独QP的三个盲区 (SBC弥补)

#### 盲区1: QP只看一步, 无法保证长期安全

CBF 约束的保证是:

$$b(s_{t+1}) \geq (1 - p \cdot dt) \cdot b(s_t)$$

这意味着即使每步都满足 CBF, 屏障值 $b(s)$ 也会以指数速率 $e^{-p\cdot t}$ 衰减。

```
单独QP的问题:
  t=0:    b=10.0  ✅ QP: "这一步安全"
  t=1s:   b=5.0   ✅ QP: "这一步安全"  
  t=2s:   b=2.5   ✅ QP: "这一步安全"
  t=3s:   b=1.25  ✅ QP: "这一步安全"
  t=4s:   b=0.625 ✅ QP: "这一步安全"
  ...
  t=10s:  b=0.05  ⚠️ QP: "这一步...勉强安全"
  t=10.1s: b<0    ❌ QP: 无解! CBF约束不可行!

问题: QP 每步都说"安全", 但系统在缓慢走向危险!
```

**SBC 的弥补**: SBC 验证的是**无限时域**的期望行为。SBC 条件 $\mathbb{E}[B(s_{t+1})] \leq B(s_t)$ 保证了系统的长期稳定性——不仅是下一步, 而是永远。

```
SBC+QP的好处:
  SBC 训练时确保: 闭环系统的 B(s) 是上鞅 (期望递减)
  → 系统不会"漂移"到危险区域
  → QP 的逐步安全有了全局保证
  → 两者共同: SBC 保证不会走到危险, QP 保证走的过程中不犯错
```

#### 盲区2: QP不知道状态有多"危险"

单独 QP 只知道: $b(s) = d - v \cdot t_{gap}$ 是否 ≥ 0。但它不知道:

- 当前状态在训练分布内还是外?
- 控制器在这个状态的历史可靠性如何?
- 从当前状态到不安全区域的"距离"有多远(概率意义上)?

**SBC 的弥补**: SBC 的 $B(s)$ 值包含了概率信息——它衡量的是 "从状态 s 出发, 未来进入不安全区域的概率上界"。

$$P(\text{未来进入不安全区域} \mid s_0 = s) \leq \frac{B(s)}{1/(1-p_{target})}$$

所以:
- $B(s)$ 很小 → SBC 认证这个状态"大概率永远安全" → QP 可以放松
- $B(s)$ 很大 → SBC 警告这个状态"可能危险" → QP 应该收紧

这就是 **SBC 调制 CBF 参数** (V3B 的设计动机):

$$p(s) = p_{min} + (p_{max} - p_{min}) \cdot \sigma\left(\frac{B(s) - B_{thresh}}{T}\right)$$

> 注: 实验中 V3B 效果不佳是因为 SBC 的 B(s) 值未校准(全在 25-35 范围), 而非思路错误。用 Barrier 函数 $b(s)$ 替代 B(s) 做调制 (V3D) 效果显著——这本质上是简化版的 SBC→CBF 信息传递。

#### 盲区3: QP无法提供"可量化的安全声明"

单独 QP 只能声明: "在每个时间步, 如果 CBF 约束有解, 则系统安全"。

但这无法回答:
- 约束**无解**的概率是多少? (QP infeasible → 没有安全控制可选)
- 系统的**整体**安全水平是多少?
- 怎么向第三方(监管机构、用户)证明系统是安全的?

**SBC 的弥补**: SBC 输出一个**具体的数字**: "96.58%"。

```
SBC + QP 的安全声明:
  "本系统:
   1. 在 96.58% 的概率下, 控制器从初始区域出发永不进入不安全区域 (SBC)
   2. 在 100%  的确定性下, 每个时间步的控制满足 CBF 安全约束 (QP)
   3. 当 CBF 约束不可行时, QP 输出松弛解并报警"
```

这是**可审计、可认证、可比较**的安全声明。单独 QP 做不到。

### 8.3 单独SBC的三个盲区 (QP弥补)

反过来, SBC 也有 QP 才能填补的漏洞:

| SBC 的盲区 | QP 的弥补 |
|-----------|----------|
| **概率保证 ≠ 绝对安全**: 96.58% 安全 = 还有 3.42% 危险 | QP 在每次危险发生时**确定性**拦截 |
| **分布外失效**: SBC 验证只在训练分布内有效, OOD 时 B(s) 无意义 | QP 只需要真实状态 d, v (物理量, 永远可靠) |
| **不能实时干预**: SBC 是离线验证工具, 不能修改运行时行为 | QP 在推理时**每步**检查和修正控制 |

**关键差异**:
- SBC 验证的是**控制器策略** → "这个策略大概率安全"
- QP 保护的是**具体动作** → "这个动作一定安全"

如果一个攻击者在推理时修改了 NN 权重:
- SBC: 验证的是原始权重 → **对攻击后的系统无效**
- QP: 仍然检查每步动作 → **独立于 NN 工作**

### 8.4 具体的协同优势

#### 优势1: "不牺牲"原则

```
单独QP (嵌入训练, 类似v2):
  训练: controller → QP → u → 动力学 → SBC 验证
  → QP 的非线性增加了闭环 Lipschitz 常数
  → SBC 验证更保守
  → 概率界从 95.8% 降到 87.9%  ❌

SBC+QP 解耦 (v3):
  训练: controller → u → 动力学 → SBC 验证 (无 QP!)
  推理: controller → u_ref → QP → u_safe
  → SBC 概率界保持 96.58%
  → 推理时仍有 QP 保护
  → 两全其美  ✅
```

**这是 v3 最核心的贡献**: 发现 "SBC 训练" 和 "QP 部署" 必须解耦, 否则 QP 会破坏 SBC 的验证质量。

#### 优势2: 互补的失效模式

```
失效场景                        SBC 能防?    QP 能防?    组合能防?
────────────────────────────────────────────────────────────
NN 在训练分布内偶尔失误           ✅ 概率界    ✅ 逐步修正   ✅✅
NN 遭遇 OOD 输入 (分布外)        ❌ 界失效    ✅ 独立工作   ✅
攻击者修改 NN 权重               ❌ 验证失效  ✅ 独立工作   ✅
CBF 参数调得太松 (p太小)         ✅ B(s)监测  ❌ 约束太弱   ✅
QP 不可行 (无安全控制)            ✅ 预期内    ⚠️ 松弛解     ✅
长期缓慢漂移到危险                ✅ 上鞅保证  ❌ 近视       ✅
传感器噪声叠加                    ❌ 界松动    ✅ 逐步过滤   ✅
```

每种机制有自己的盲区, 但两者的盲区**不重叠**——这是互补的理想状态。

#### 优势3: 开发流程的分离

```
SBC 团队 (验证):                 QP 团队 (防护):
  "我们证明控制器在                "我们在控制器外面加一个
   96.58% 概率下安全"              安全过滤器"

  工作:                           工作:
  - 训练 SBC 网络                  - 设计 CBF 约束
  - IBP 验证                       - 调 p 参数
  - 计算概率界                     - 测试干预率
  - 提供安全证书                   - 部署实时检查

  交付: "安全证书"                 交付: "安全过滤器"

           └──────────┬──────────┘
              集成: 双重安全系统
```

两个模块可以**独立开发、独立验证、独立更新**——这是软件工程的理想架构。

### 8.5 实验数据中的体现

虽然实验未能展示 QP 改变违例率 (因为 PPO 太安全), 但数据仍然体现了 SBC+QP 的协同:

| 指标 | 单独 SBC | 单独 QP | **SBC + QP (v3)** |
|------|---------|---------|-------------------|
| 形式化概率界 | 96.58% | N/A (无法提供) | **96.58%** ✅ |
| 逐步确定性安全 | N/A (不干预) | ✅ | ✅ |
| 噪声响应 (干预率) | N/A | 21%→100% | **21%→100%** (自适应) |
| 模块化 | — | — | **解耦: 分别训练/部署** |
| 可认证性 | 概率声明 | 逐步声明 | **两者都有** |

### 8.6 类比: 为什么飞机需要两个引擎

```
单引擎飞机 (单独QP):
  ✅ 轻便
  ❌ 引擎故障 → 坠毁
  
双引擎飞机 (SBC + QP):
  ✅ 一个引擎故障 → 另一个继续工作
  ✅ 两个引擎同时故障的概率 << 单个故障概率
  ❌ 稍微重一点 (实现复杂度)

SBC = 左引擎 (概率验证)
QP  = 右引擎 (确定性防护)
```

在安全关键系统中, **冗余不是浪费, 是必须**。SBC 和 QP 提供的是**异质冗余**——它们用不同的机制防止不同的问题, 比两个相同机制的组合更强大。

### 8.7 总结: SBC+QP 的价值公式

```
SBC + QP 的价值 = 
    SBC的全局概率保证
  + QP的局部确定性保证  
  + 两者的盲区互补
  + 解耦带来的模块化
  - 额外的实现复杂度
  ───────────────────
  = 可认证的、分层的、完整的可信安全系统
```

**一句话**: 单独 QP 只能保证 "每步不出错", SBC+QP 能保证 "系统整体大概率安全 + 每步确定不出错"。


## 9. 实验结论

### 9.1 核心结论

1. **指标区分很重要**: CBF约束违反 (margin<0) **确实存在 1-2%**, QP 通过合理调参可将违反率降至 0%。Barrier变负 (b_next<0) 在单步测试中总是 0% (dt=0.05 物理限制)。之前报告混淆了这两个指标。

2. **V3A p=2.0 和 V3D p_min=0.5 达到 100% CBF 安全率** — QP 通过合理参数可完全消除 CBF 约束违反。

3. **受控实验证明 QP 机制有效**: 坏控制器 100% 被 QP 阻止, Baseline 1-3 秒内必违例。

4. **QP + SBC 是互补的双重安全**: SBC 给全局概率界 (96.6%), QP 给局部确定性保证 (CBF 100% 安全率)。

### 9.2 QP 加进去的好处总结

| 好处 | 数据支撑 | 实验 |
|------|---------|------|
| **防止 NN 故障导致的事故** | 坏控制器: 0/3 Baseline 安全 vs 3/3 QP 安全 | 受控验证 |
| **自适应风险响应** | 噪声 0→7: 干预率 21%→100% (线性正相关) | 实验一 |
| **最小侵入性** | 21% 干预率 (正常), 只在必要时修正 | 实验一 |
| **SBC 不受干扰** | 96.58% 概率界保持 (vs v2 的 87.9%) | SBC 训练 |
| **可认证的确定性保证** | CBF 数学约束保证每步安全 | 理论 |
| **模块化可插拔** | 零训练, 插入任何控制器即可工作 | 架构 |

### 8.3 局限与展望

| 局限 | 改进方向 |
|------|---------|
| PPO 太强, 遮住了 QP 的价值 | 使用更弱/更激进的控制器对比 |
| AEBS 动力学太慢 (dt=0.05) | 增大 dt 或使用更动态的场景 |
| 单步检查只保证了一步 | 扩展到多步 CBF 或 MPC-CBF |
| 感知错误不影响结果 | 使用更大的感知扰动或完全移除感知 |
| 1D 控制太简单 | 扩展到 2D (转向+加速) |
